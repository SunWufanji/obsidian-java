# 网络层

## 1. IP 协议是什么？

IP（互联网协议）是网络层的核心协议，负责把数据包从源主机送到目的主机。就像寄快递时写的收件地址，IP 地址就是互联网上每台设备的"地址"。

**IPv4**：32 位地址，用点分十进制表示，如 `192.168.1.1`，总共约 43 亿个地址（已耗尽）。

**IPv6**：128 位地址，用冒号十六进制表示，如 `2001:db8::1`，地址数量约 3.4×10^38 个，足够用。

IPv4 vs IPv6 头部对比：
- IPv4 头部：20-60 字节，有校验和、头部可变长度
- IPv6 头部：40 字节固定，去掉了校验和（交给上层处理），简化了路由转发

## 2. ICMP 是什么？

ICMP（互联网控制报文协议）是 IP 的"助手"，用来报告错误和传递控制信息。

**常见用途**：
- **ping 命令**：用 ICMP Echo Request/Reply 测试网络连通性
- **traceroute**：用 ICMP 超时报文追踪数据包经过的路由
- **目的不可达**：路由器转发失败时通知发送方

**ping 的过程**：发送 ICMP Echo Request → 接收方回复 ICMP Echo Reply → 根据往返时间判断网络质量。

## 3. DHCP 是什么？

DHCP（动态主机配置协议）就是让你连上网络后**自动获取 IP 地址**，不用手动配置。

想象你去酒店入住，不需要自己带床铺和家具，酒店自动给你安排好房间（IP 地址）、钥匙（子网掩码）、路由器地址（网关）、DNS 服务器地址。

DHCP 的工作流程（DORA）：
1. **Discover**（发现）：客户端广播"有没有 DHCP 服务器？"
2. **Offer**（提供）：DHCP 服务器回复"我这里有 IP 可以给你用"
3. **Request**（请求）：客户端说"我要这个 IP"
4. **Acknowledge**（确认）：服务器说"好，这个 IP 归你了，租期 24 小时"

## 4. NAT 是什么？

NAT（网络地址转换）就是把内网的私有 IP 地址转换成公网 IP 地址，让内网设备能访问互联网。

**为什么需要 NAT？** IPv4 地址不够用，一个家庭/公司有几十上百台设备，不可能每个都配一个公网 IP。NAT 让所有设备共享一个公网 IP。

**NAT 的工作原理**：
- 内网设备 `192.168.1.100:5000` 要访问百度
- NAT 路由器把源地址改成公网 IP `203.0.113.1:40000`
- 百度回复给 `203.0.113.1:40000`
- NAT 路由器查表，把目标地址改回 `192.168.1.100:5000`

**NAT 穿透问题**：P2P 通信时，两台内网设备无法直接通信（都有私有 IP），需要打洞（STUN/TURN）技术。

## 5. 路由选择是怎样的？

路由就是数据包从源到目的地"走哪条路"的问题。

**路由表**：每个路由器都有一张路由表，记录"去某个网络应该往哪个方向转发"。

**路由选择方式**：
- **静态路由**：管理员手动配置（适合小网络）
- **动态路由**：路由器之间自动学习和更新路由表（适合大网络）

**常见路由协议**：
- **RIP**：基于跳数，最多 15 跳，适合小网络
- **OSPF**：基于链路状态，计算最短路径，适合企业内网
- **BGP**：互联网骨干网使用的协议，AS（自治系统）之间的路由

**面试简答版：**
> 路由就是数据包转发的路径选择。小网络用静态路由，大网络用动态路由协议（OSPF、BGP）。每个路由器维护一张路由表，根据目标 IP 查表决定下一跳往哪走。

## 6. IPv6 相比 IPv4 有哪些主要优势？

IPv6 是 IPv4 的升级版，主要解决了 IPv4 地址耗尽的问题，同时做了很多改进。

### 地址空间

| 对比项 | IPv4 | IPv6 |
|-------|------|------|
| 地址长度 | 32 位 | 128 位 |
| 地址数量 | 约 43 亿（已耗尽） | 约 3.4×10^38（用不完） |
| 表示方式 | 点分十进制 `192.168.1.1` | 冒号十六进制 `2001:db8::1` |

IPv6 的地址数量大到地球上的每一粒沙子都能分配一个 IP 地址。

### 头部简化

IPv4 头部 20-60 字节（不固定），IPv6 头部固定 40 字节。去掉了校验和字段（交给上层处理），路由器转发时不需要重新计算校验和，**转发效率更高**。

### 内建安全性

IPv6 设计时就把 **IPsec**（网络层安全协议）集成进去了，提供内建的加密和认证。IPv4 的安全是后加的，不是标配。

### 自动配置（SLAAC）

IPv6 支持**无状态地址自动配置**——设备接入网络后自动生成 IP 地址，不需要 DHCP 服务器。IPv4 要么手动配 IP，要么靠 DHCP 服务器分配。

### 更好的移动性

IPv6 设备在不同网络间切换时，可以保持网络连接不断。IPv4 切换网络通常要换 IP，连接就断了。

### 改进的路由

IPv6 地址的层次结构更清晰，路由器可以做**聚合路由**，路由表更小、查找更快。

### 对比总结

| 优势 | 说明 |
|------|------|
| 海量地址 | 128 位，不怕不够用 |
| 头部简化 | 固定 40 字节，转发更快 |
| 内建安全 | IPsec 集成，加密认证标配 |
| 自动配置 | SLAAC 无需 DHCP |
| 移动性好 | 切换网络连接不断 |
| 路由高效 | 地址层次清晰，聚合路由 |

**面试简答版：**
> IPv6 六大优势：128 位海量地址解决耗尽问题、头部固定 40 字节转发更快、内建 IPsec 安全加密、SLAAC 自动配置无需 DHCP、移动性好切换不断连、路由聚合效率高。
