# 数据链路层

## 1. ARP 协议是什么？

ARP（地址解析协议）就是把 **IP 地址转换成 MAC 地址**。

IP 地址是网络层的"门牌号"，MAC 地址是数据链路层的"身份证号"。在局域网内通信时，光知道 IP 地址还不够，得知道对方的 MAC 地址才能封装成帧发出去。

**ARP 的工作过程**：
1. 主机 A 想给同网段的主机 B（已知 IP，不知道 MAC）发数据
2. A 广播一个 ARP 请求："谁的 IP 是 192.168.1.2？请告诉我你的 MAC 地址"
3. 所有主机都收到这个广播，但只有 B 会回复："我是 192.168.1.2，我的 MAC 地址是 xx:xx:xx"
4. A 把这个 IP-MAC 对应关系缓存起来（ARP 缓存），下次直接用

**ARP 缓存**：每台主机都有一个 ARP 缓存表，存储局域网内 IP 和 MAC 的映射，有有效期（通常几分钟）。

---

## 2. RARP 协议是什么？

RARP（反向地址解析协议）是 ARP 的反向操作：**已知 MAC 地址，求 IP 地址**。

现在基本被 DHCP 取代了，因为 DHCP 不仅能分配 IP，还能分配子网掩码、网关、DNS 等信息，功能更全面。

---

## 3. MAC 地址是什么？

MAC 地址（媒体访问控制地址）是网卡的**物理地址**，全球唯一，用 12 位十六进制表示，如 `00:1A:2B:3C:4D:5E`。

**MAC 地址 vs IP 地址**：
- MAC 地址：硬件层面的标识，不会变（除非换了网卡）
- IP 地址：网络层面的标识，换了网络就会变

类比：MAC 地址是你的身份证号（终身不变），IP 地址是你当前的住址（搬家就变了）。

**面试简答版：**
> MAC 地址是网卡的物理标识，全球唯一，用于局域网内设备间的通信。IP 地址用于跨网络寻址，MAC 地址用于同一网络内帧的传输。
