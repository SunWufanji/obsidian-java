# CDN

## 1. CDN 是什么？

CDN（内容分发网络）就是把网站的"热门内容"提前复制到离你最近的服务器上，你访问时就近获取，不用跑到源服务器去拿。

打个比方：你在老家（比如成都）想喝星巴克，如果没有 CDN，你得跑到上海总部去买（源服务器）。有了 CDN，成都就有星巴克分店（边缘节点），你走两步就到了，体验一模一样。

**解决什么问题？** 用户分布在全国甚至全球，如果所有人都访问同一台服务器：
- 远距离传输延迟高
- 服务器压力大
- 一个机房挂了全挂

---

## 2. CDN 的工作原理是怎样的？

当你访问一个使用了 CDN 的网站时：

```
用户 → DNS解析 → CDN的智能DNS → 就近的边缘节点 → 有缓存直接返回 / 没缓存回源站获取
```

详细流程：

1. **用户请求**：浏览器发起请求，DNS 解析时发现域名配了 CNAME 指向 CDN
2. **智能 DNS**：CDN 的 DNS 根据用户的 IP 地址、运营商、网络状况，选择最优的边缘节点
3. **就近访问**：用户被引导到最近的 CDN 边缘节点
4. **缓存命中**：如果边缘节点有缓存，直接返回给用户（速度快）
5. **回源**：如果缓存没有或过期，CDN 节点去源服务器取内容，缓存后再返回给用户

**核心思想**：用空间换时间，把内容放到离用户最近的地方。

## 3. CDN 的常见应用场景

| 场景 | 说明 |
|------|------|
| 静态资源加速 | JS、CSS、图片、视频等文件缓存到 CDN |
| 网页加速 | HTML 页面的加速 |
| 视频点播/直播 | 大流量场景，CDN 分担压力 |
| 下载加速 | 游戏安装包、软件更新等大文件下载 |
| 安全防护 | DDoS 防护、WAF（Web 应用防火墙） |

---

## 4. CDN 缓存策略

CDN 不是什么都缓存，有一套策略：

- **静态资源**：JS、CSS、图片等几乎不变的文件，缓存时间长（可能几天）
- **HTML 页面**：缓存时间短或不缓存，因为内容经常变
- **动态内容**：不缓存（如 API 接口返回的数据）

缓存过期策略：
- **TTL 过期**：缓存到期后重新从源站获取
- **主动更新**：源站内容变化时主动通知 CDN 刷新缓存

**面试简答版：**
> CDN 把内容缓存到离用户最近的节点，用户就近获取，减少延迟和源站压力。核心是智能 DNS 调度 + 缓存机制，适合静态资源和大流量场景。
