# 计算机网络体系结构

## 1. OSI 七层模型是什么？
![[Pasted image 20260522200502.png]]
想象你要给远方的朋友寄一本书。你把书装进信封（**应用层**），信封装进包裹（**表示层**），包裹贴上快递单（**会话层**），快递公司决定走哪条路线（**传输层**），中途经过多个中转站（**网络层**），每个中转站内部有分拣机器在传送带上处理包裹（**数据链路层**），传送带本身就是物理线路（**物理层**）。

OSI 把网络通信分成了 7 层，每层各司其职：

| 层 | 名称 | 干什么的 | 举例 |
|---|------|---------|------|
| 7 | 应用层 | 直接为用户服务，提供接口 | HTTP、FTP、SMTP、DNS |
| 6 | 表示层 | 数据格式转换、加密解密 | SSL/TLS、JPEG编码 |
| 5 | 会话层 | 建立和管理连接（会话） | NetBIOS、RPC |
| 4 | 传输层 | 端到端的可靠传输 | TCP、UDP |
| 3 | 网络层 | 路由选择、IP寻址 | IP、ICMP、路由协议 |
| 2 | 数据链路层 | 帧的传输、差错检测 | 以太网、MAC地址 |
| 1 | 物理层 | 比特流在物理介质上传输 | 光纤、网线、无线电波 |

**面试回答模板：**
> OSI 七层模型从下到上分别是物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。其中物理层负责比特流传输，数据链路层负责帧传输和差错检测，网络层负责路由和寻址，传输层负责端到端通信，上面三层主要为应用程序服务。

## 2. OSI 七层模型和 TCP/IP 四层模型有什么区别？

OSI 是理论上的"完美模型"，但太复杂，实际没人用。TCP/IP 是互联网真正在用的协议栈，它把 OSI 的上面三层合并成"应用层"，把数据链路层和物理层合并成"网络接口层"。

| OSI 七层 | TCP/IP 四层 | TCP/IP 对应协议 |
|---------|------------|----------------|
| 应用层 | 应用层 | HTTP、FTP、DNS、SMTP |
| 表示层 | ↑ | ↑ |
| 会话层 | ↑ | ↑ |
| 传输层 | 传输层 | TCP、UDP |
| 网络层 | 网络层 | IP、ICMP、ARP |
| 数据链路层 | 网络接口层 | 以太网、Wi-Fi |
| 物理层 | ↑ | ↑ |

**面试回答模板：**
> OSI 是七层理论模型，实际中用的是 TCP/IP 四层模型。TCP/IP 把 OSI 的应用层、表示层、会话层合并为应用层，数据链路层和物理层合并为网络接口层。TCP/IP 是互联网实际使用的协议栈。

## 3. TCP/IP 参考模型各个组件的含义和主要功能

TCP/IP 四层模型中每一层都有对应的协议，面试常问"简单介绍一下各层的主要协议和功能"。

**应用层**：直接和用户打交道。HTTP 用于网页浏览，DNS 用于域名解析，FTP 用于文件传输，SMTP 用于发邮件。这一层的数据单位叫"报文"。

**传输层**：负责端到端（进程到进程）的通信。TCP 提供可靠传输，有连接、有确认、有重传；UDP 不可靠但快，适合视频、游戏等场景。数据单位叫"段"（TCP）或"用户数据报"（UDP）。

**网络层**：负责主机到主机的通信，核心是 IP 协议，通过 IP 地址寻址，通过路由器转发数据包。数据单位叫"分组"或"数据包"。

**网络接口层**：负责在物理网络上传输数据，处理 MAC 地址、帧的封装和差错检测。包括以太网、Wi-Fi 等。

**面试回答模板：**
> TCP/IP 四层模型从上到下是：应用层提供各种网络应用服务如 HTTP、DNS；传输层负责端到端通信，TCP 可靠但慢，UDP 快但不可靠；网络层通过 IP 协议实现主机间的寻址和路由；网络接口层负责物理网络上的帧传输。

## 4. 为什么要分层？

想象一家公司，CEO 不可能直接去仓库搬货。他只需要告诉副总，副总告诉部门经理，经理告诉仓库管理员。每一层只和相邻层打交道，改了仓库的搬运方式，CEO 完全不用知道。

分层的好处：
- **各层独立**：改一层不影响其他层，比如换光纤（物理层）不用改 HTTP 协议（应用层）
- **职责清晰**：每层专注自己的事，便于理解和实现
- **灵活扩展**：新增功能只需在对应层添加，比如 HTTPS 就是在应用层加了 TLS 加密

**面试简答版：**
> 分层是为了让各层职责独立、互不干扰。改底层实现不需要动上层逻辑，就像公司分工一样，各司其职提高效率。
