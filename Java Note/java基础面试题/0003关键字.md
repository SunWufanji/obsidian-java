# 关键字

## 1. final、finally、finalize 的区别？

#### 先给结论
- `final` 是关键字，用来修饰类、方法、变量。
- `finally` 是异常处理结构的一部分，通常配合 `try-catch` 使用。
- `finalize()` 是 `Object` 类里的一个方法，和 GC 相关，但现在已经不推荐使用了。

#### 核心解释
- `final`
  - 修饰变量：值不能再次赋值。
  - 修饰方法：不能被子类重写。
  - 修饰类：不能被继承。
- `finally`
  - 写在 `try-catch-finally` 结构里。
  - 一般用于释放资源，比如关闭流、关闭连接。
  - 不管是否发生异常，通常都会执行。
- `finalize()`
  - 是对象在被垃圾回收前，JVM 可能调用的方法。
  - 重点是“可能”，调用时机不确定，甚至可能根本不调用。
  - 所以不能把重要资源释放逻辑放在这里。

#### 简单代码示例
```java
public class Demo {
    public static void main(String[] args) {
        final int a = 10;

        try {
            System.out.println("try");
        } finally {
            System.out.println("finally");
        }

        System.out.println(a);
    }
}
```

#### 输出结果
```text
try
finally
10
```

#### 输出 / 现象解释
- `a` 被 `final` 修饰后，后面不能再重新赋值。
- `finally` 会在 `try` 执行后继续执行，所以打印了 `finally`。

#### 面试易错点 / 补充点
- `final` 和 `finally` 完全不是一回事，一个是关键字修饰符，一个是异常处理结构。
- `final` 修饰引用变量时，不能改“指向”，但对象内部状态可能还能改。
- `finalize()` 不适合做资源回收，现在面试里最好明确说“不推荐依赖它”。

#### 面试回答模板
- `final`、`finally`、`finalize` 容易混，但它们不是一类东西。`final` 是关键字，可修饰类、方法、变量；`finally` 是异常处理里的代码块，通常用于资源释放；`finalize()` 是 `Object` 的方法，和垃圾回收有关，但调用时机不确定，现在已经不推荐依赖它。

#### 面试简答版
- `final`：修饰类、方法、变量。
- `finally`：异常处理里的收尾代码块。
- `finalize()`：GC 相关方法，不可靠，不推荐使用。

## 2. 为什么要用 static 关键字？

#### 先给结论
- `static` 的核心作用是：让成员属于“类”而不是“对象”。
- 用了 `static` 之后，不需要创建对象，也可以通过类名直接访问。
- 它常用于共享数据、工具方法、常量、程序入口。

#### 核心解释
- 普通成员属于对象，每创建一个对象，就有自己的一份。
- 静态成员属于类，在内存里通常只保留一份，所有对象共享。
- 所以 `static` 很适合表示“所有对象都共用的东西”。

#### 常见使用场景
##### 1）共享变量
- 比如统计一共创建了多少个对象。

##### 2）工具方法
- 比如 `Math.max()`、`Arrays.sort()` 这种方法，不需要依赖某个具体对象。

##### 3）常量
- 常配合 `final` 一起使用，写成 `public static final`。

##### 4）程序入口
- `main` 方法必须是 `static`，这样 JVM 不需要先创建对象就能调用。

#### 简单代码示例
```java
class Student {
    static int count = 0;

    public Student() {
        count++;
    }
}

public class Demo {
    public static void main(String[] args) {
        new Student();
        new Student();
        new Student();

        System.out.println(Student.count);
    }
}
```

#### 输出结果
```text
3
```

#### 输出 / 现象解释
- `count` 是静态变量，属于 `Student` 类本身。
- 每创建一个对象，都会让这同一份 `count` 加 1，所以最后是 3。

#### 面试易错点 / 补充点
- `static` 不代表“生命周期短”，恰恰相反，它通常跟类一起存在。
- 静态成员是共享的，所以并发场景下要注意线程安全。
- 工具类常把构造方法私有化，防止被实例化。

#### 面试回答模板
- 使用 `static` 的原因，本质上是为了让成员归属于类而不是对象。这样既可以让多个对象共享同一份数据，也可以让一些不依赖实例的方法直接通过类名调用，所以 `static` 常用于共享变量、工具方法、常量和 `main` 方法。

#### 面试简答版
- `static` 表示类级别成员。
- 可共享、可用类名直接访问。
- 常用于共享变量、工具方法、常量、`main` 方法。

## 3. “static” 关键字是什么意思？Java 中是否可以覆盖（override）一个 private 或者 static 的方法？

#### 先给结论
- `static` 表示“类级别”的成员，属于类，不属于某个对象。
- `private` 方法不能被重写。
- `static` 方法也不能被重写，只能被“隐藏”。

#### 核心解释
##### 1）为什么 `private` 不能被重写
- 重写的前提是：子类能“看见”父类这个方法。
- `private` 方法只能在父类自己内部访问，子类根本看不见。
- 所以子类写一个同名同参的方法，也只是“新方法”，不是重写。

##### 2）为什么 `static` 不能被重写
- 重写依赖多态，核心是“运行时根据对象实际类型决定调用哪个方法”。
- 但 `static` 方法属于类，不属于对象。
- 它的调用在编译期就能确定，更接近“类名绑定”，不是动态分派。
- 所以子类写同名同参 `static` 方法，只是把父类方法“隐藏”了。

#### 简单代码示例
```java
class Parent {
    static void show() {
        System.out.println("parent");
    }
}

class Child extends Parent {
    static void show() {
        System.out.println("child");
    }
}

public class Demo {
    public static void main(String[] args) {
        Parent p = new Child();
        Child c = new Child();

        p.show();
        c.show();
    }
}
```

#### 输出结果
```text
parent
child
```

#### 输出 / 现象解释
- `p` 虽然指向的是 `Child` 对象，但 `p.show()` 调用的还是 `Parent.show()`。
- 这说明 `static` 方法不是按对象实际类型走多态，而是按“引用变量的编译期类型”决定。
- 所以这里不是方法重写，而是方法隐藏。

#### 面试易错点 / 补充点
- `private` 方法、`static` 方法、`final` 方法都不属于正常意义上的可重写方法。
- 不要把“重载”和“重写”混了：这里讨论的是 override，不是 overload。
- 调用静态方法建议用类名调用，不要用对象调用，虽然语法上很多时候能写，但容易误导。

#### 面试回答模板
- `static` 表示成员属于类本身。Java 里 `private` 方法不能被重写，因为子类不可见；`static` 方法也不能被重写，因为它不参与运行时多态，只能在子类中定义同名同参方法去隐藏父类的静态方法。

#### 面试简答版
- `static`：类级别成员。
- `private`：不能重写。
- `static`：不能重写，只能隐藏。

## 4. 是否可以在 static 环境中访问非 static 变量？

#### 先给结论
- 不能直接访问。
- 如果一定要访问，必须先有对象，再通过对象访问。

#### 核心解释
- `static` 环境属于类级别，比如静态方法、静态代码块。
- 而非静态变量属于对象级别。
- 类加载时不一定已经有对象，所以静态环境里没有天然的 `this`。
- 没有对象，就不知道该访问哪一个实例变量。

#### 简单代码示例
```java
public class Demo {
    int age = 18;
    static int count = 1;

    static void test() {
        // System.out.println(age); // 编译报错
        Demo d = new Demo();
        System.out.println(d.age);
        System.out.println(count);
    }

    public static void main(String[] args) {
        test();
    }
}
```

#### 输出结果
```text
18
1
```

#### 输出 / 现象解释
- `age` 是实例变量，不能在静态方法里直接访问。
- 创建对象 `d` 后，就能通过 `d.age` 访问。
- `count` 是静态变量，属于类，所以能直接访问。

#### 面试易错点 / 补充点
- 不是“静态环境永远不能访问实例变量”，而是“不能直接访问”。
- 只要拿到对象，就能访问对应对象的实例变量。

#### 面试回答模板
- 在 `static` 环境中不能直接访问非 `static` 变量，因为静态环境属于类，没有 `this`，而实例变量属于对象，必须先创建对象或者拿到对象引用，再通过对象访问。

#### 面试简答版
- 不能直接访问。
- 想访问就先拿对象。

## 5. static 静态方法能不能引用非静态资源？

#### 先给结论
- 不能直接引用非静态资源。
- 这里的“非静态资源”既包括实例变量，也包括实例方法。

#### 核心解释
- 静态方法没有 `this`。
- 没有 `this`，就意味着它不知道当前到底是哪一个对象。
- 而实例资源必须依附具体对象存在，所以静态方法不能直接使用它们。

#### 简单代码示例
```java
public class Demo {
    int age = 18;

    void run() {
        System.out.println("run: " + age);
    }

    static void test() {
        // System.out.println(age); // 编译报错
        // run(); // 编译报错

        Demo d = new Demo();
        System.out.println(d.age);
        d.run();
    }

    public static void main(String[] args) {
        test();
    }
}
```

#### 输出结果
```text
18
run: 18
```

#### 输出 / 现象解释
- 在静态方法 `test()` 里，实例变量 `age` 和实例方法 `run()` 都不能直接调用。
- 但只要先创建对象 `d`，就能通过对象访问。

#### 面试易错点 / 补充点
- 这题和上一题本质一样，只是上一题强调“非静态变量”，这题强调“非静态资源”。
- “资源”范围更大，除了变量，还包括普通方法。

#### 面试回答模板
- 静态方法不能直接引用非静态资源，因为静态方法属于类，没有 `this`，而实例变量和实例方法都依赖具体对象。要使用的话，必须先拿到对象，再通过对象调用。

#### 面试简答版
- 不能直接引用。
- 实例变量、实例方法都得通过对象访问。

## 6. static 静态方法里面能不能引用静态资源？

#### 先给结论
- 可以，而且可以直接引用。

#### 核心解释
- 静态方法和静态变量、静态方法本身都属于类级别。
- 它们处于同一个层面，所以可以直接访问。
- 同一个类中通常可以直接写成员名，不同类中一般通过“类名.成员”访问。

#### 简单代码示例
```java
public class Demo {
    static int count = 10;

    static void show() {
        System.out.println("show");
    }

    static void test() {
        System.out.println(count);
        show();
    }

    public static void main(String[] args) {
        test();
    }
}
```

#### 输出结果
```text
10
show
```

#### 输出 / 现象解释
- `test()`、`count`、`show()` 都是静态的，属于同一个类级别，所以可以直接访问。

#### 面试易错点 / 补充点
- 这里说“可以直接访问”，前提通常是同一个类里。
- 如果是别的类的静态资源，推荐写成 `类名.静态成员`，可读性更清楚。

#### 面试回答模板
- 静态方法里面可以直接引用静态资源，因为它们都属于类级别成员，不依赖对象实例，所以同一个类中可以直接访问，不同类中通常通过类名访问。

#### 面试简答版
- 可以。
- 因为都是类级别资源。

## 7. 非静态方法里面能不能引用静态资源？

#### 先给结论
- 可以，而且也能直接引用。

#### 核心解释
- 非静态方法是属于对象的，但对象天然知道自己属于哪个类。
- 类级别的资源对对象来说也是可见的。
- 所以在实例方法里，既能访问实例资源，也能访问静态资源。

#### 简单代码示例
```java
public class Demo {
    static int count = 10;
    int age = 18;

    void test() {
        System.out.println(count);
        System.out.println(age);
    }

    public static void main(String[] args) {
        new Demo().test();
    }
}
```

#### 输出结果
```text
10
18
```

#### 输出 / 现象解释
- `test()` 是实例方法，它既能访问类共享的 `count`，也能访问当前对象自己的 `age`。

#### 面试易错点 / 补充点
- 非静态方法权限更“宽”一些，它既能碰静态资源，也能碰实例资源。
- 但要注意：访问静态变量时，本质上访问的还是类共享数据，不是对象私有数据。

#### 面试回答模板
- 非静态方法可以引用静态资源，因为实例方法既有对象上下文，也能访问所属类的类级别成员。所以实例方法里通常既能访问静态变量和静态方法，也能访问实例变量和实例方法。

#### 面试简答版
- 可以。
- 实例方法既能访问实例资源，也能访问静态资源。

## 8. Java 静态变量、代码块、和静态方法的执行顺序是什么？

#### 先给结论
- 类加载并初始化时，会先执行静态变量赋值和静态代码块。
- 如果它们交叉出现，就按照源码中的书写顺序执行。
- 静态方法不会因为“定义在那里”就自动执行，只有被调用时才执行。

#### 核心解释
- 很多人容易把“静态方法”也算进自动初始化流程里，这是不对的。
- 真正自动执行的是：
  - 静态变量的显式赋值
  - 静态代码块
- 它们在类初始化阶段执行，而且是按源码顺序执行。
- 静态方法只是类加载后“可以被调用”，但不会自动跑。

#### 简单代码示例
```java
public class Demo {
    static int a = print("1. 静态变量 a");

    static {
        System.out.println("2. 静态代码块");
    }

    static int b = print("3. 静态变量 b");

    static int print(String msg) {
        System.out.println(msg);
        return 0;
    }

    static void show() {
        System.out.println("4. 静态方法 show");
    }

    public static void main(String[] args) {
        show();
    }
}
```

#### 输出结果
```text
1. 静态变量 a
2. 静态代码块
3. 静态变量 b
4. 静态方法 show
```

#### 输出 / 现象解释
- `a` 先初始化，所以先打印第 1 行。
- 然后执行静态代码块。
- 再初始化 `b`。
- 最后 `main` 方法里调用 `show()`，静态方法这时才执行。

#### 运行方式简要解释
##### 1）类初始化阶段
- 执行静态变量赋值和静态代码块。

##### 2）执行顺序
- 多个静态变量和多个静态代码块混在一起时，按源码出现顺序来。

##### 3）静态方法
- 不自动执行。
- 只有在 `main` 或其他地方显式调用后才会执行。

#### 面试易错点 / 补充点
- 不要背成“静态变量 -> 静态代码块 -> 静态方法”这种死顺序。
- 更准确的说法是：
  - 静态变量赋值和静态代码块按源码顺序执行；
  - 静态方法只有被调用时才执行。
- 如果静态方法调用触发了类初始化，那么一定是“先初始化类，再执行静态方法体”。

#### 面试回答模板
- Java 类初始化时，会先执行静态变量的显式赋值和静态代码块，而且如果两者交叉出现，就按源码书写顺序执行。静态方法本身不会自动执行，只有在被调用时才会执行；如果调用静态方法触发类初始化，那么也是先完成类初始化，再进入静态方法体。

#### 面试简答版
- 静态变量赋值、静态代码块：类初始化时按源码顺序执行。
- 静态方法：不会自动执行，调用时才执行。
