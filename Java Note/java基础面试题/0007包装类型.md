# 包装类型

## 1. 包装类型是什么？`int` 和 `Integer` 有什么区别？

#### 先给结论
- 包装类型就是把 Java 的基本类型封装成对象后的类型。
- 8 个基本类型分别对应 8 个包装类型：
  - `byte` -> `Byte`
  - `short` -> `Short`
  - `int` -> `Integer`
  - `long` -> `Long`
  - `float` -> `Float`
  - `double` -> `Double`
  - `char` -> `Character`
  - `boolean` -> `Boolean`

#### 为什么会有包装类型
- Java 是面向对象语言，很多地方要求传入对象，而不是基本类型。
- 比如集合、泛型、反射、序列化等场景都需要对象。
- 所以 Java 提供了包装类型，让基本类型也能以“对象”的形式使用。

```java
import java.util.ArrayList;
import java.util.List;

public class Demo {
    public static void main(String[] args) {
        List<Integer> list = new ArrayList<>();
        list.add(10);
        System.out.println(list.get(0));
    }
}
```

输出结果：
```java
10
```

解释：
- `List` 里只能放对象类型。
- 这里的 `10` 最终是以 `Integer` 的形式放进集合的。

#### `int` 和 `Integer` 的核心区别

### 1）本质不同
- 基本类型是 JVM 内置的数据类型，直接存值存在栈。
- 包装类型是引用类型，本质上是类、是对象存在堆。

```java
int a = 10;
Integer b = 10;
```

- `a` 是基本类型变量。
- `b` 是引用变量，指向 `Integer` 对象。

### 2）默认值不同
- 成员变量中的基本类型默认值不是 `null`。
- 成员变量中的包装类型默认值是 `null`。

```java
public class Demo {
    int a;
    Integer b;

    public static void main(String[] args) {
        Demo d = new Demo();
        System.out.println(d.a);
        System.out.println(d.b);
    }
}
```

输出结果：
```java
0
null
```

### 3）能否表示 `null`
- 基本类型不能表示 `null`。
- 包装类型可以表示 `null`，因为它本质上是引用。

```java
Integer x = null;
// int y = null; // 编译错误
```

### 4）使用场景不同
- 基本类型适合数值计算，性能更高。
- 包装类型适合需要对象的场景，比如集合、泛型、框架传参等。

### 5）性能不同
- 基本类型通常性能更高，内存开销更小。
- 包装类型因为是对象，通常会有额外的对象开销。
- 高频计算场景下优先使用基本类型。

### 6）方法能力不同
- 基本类型不是对象，不能直接调用工具方法。
- 包装类型是类，可以调用很多方法，比如：
  - `Integer.parseInt()`
  - `Integer.valueOf()`
  - `Integer.compare()`

```java
public class Demo {
    public static void main(String[] args) {
        String s = "123";
        int x = Integer.parseInt(s);
        System.out.println(x + 1);
    }
}
```

输出结果：
```java
124
```

#### `int` 和 `Integer` 比较时要注意什么
- `int` 和 `Integer` 用 `==` 比较时，通常会触发自动拆箱。
- 也就是说，`Integer` 会先拆成 `int`，再进行数值比较。
- 所以这类比较，比较的是值，不是对象地址。

```java
int a = 10000;
Integer b = new Integer(10000);
Integer c = 10000;

System.out.println(a == b);
System.out.println(a == c);
```

输出结果：
```java
true
true
```

解释：
- `a == b` 时，`b` 会自动拆箱成 `int`，再和 `a` 比较值。
- `a == c` 时，`c` 也会自动拆箱。
- 所以这里比较的是数值 `10000` 和 `10000`，结果都为 `true`。

#### 为什么集合里要用包装类型
- Java 泛型不能直接使用基本类型。
- 所以像 `List<int>` 是非法的，必须写成 `List<Integer>`。

```java
// List<int> list = new ArrayList<>(); // 编译错误
List<Integer> list = new ArrayList<>();
```

#### 什么时候用基本类型，什么时候用包装类型

### 优先用基本类型的场景
- 普通数值计算
- 循环计数
- 高频性能敏感代码
- 明确不会出现 `null` 的场景

### 优先用包装类型的场景
- 集合、泛型
- 需要表示“空值”
- 需要对象语义
- 需要调用包装类工具方法

#### 面试常见误区
- 包装类型不是基本类型，它是引用类型。
- 包装类型可以为 `null`，基本类型不行。
- `String` 不是包装类型，它是普通引用类型。

#### 面试回答模板
- 包装类型就是基本类型对应的对象类型，比如 `int` 对应 `Integer`，`double` 对应 `Double`。
- 以 `int` 和 `Integer` 为例，`int` 是基本类型，直接存值；`Integer` 是引用类型，本质上是对象，可以为 `null`。
- `int` 更适合数值计算，性能更高；`Integer` 更适合集合、泛型和需要对象的场景。
- 另外，`int` 和 `Integer` 用 `==` 比较时通常会发生自动拆箱，最终比较的是值。

#### 面试简答版
- 包装类型是基本类型对应的对象类型。
- 基本类型直接存值，性能高；包装类型是对象，可以为 `null`。
- 集合和泛型里通常必须用包装类型。

## 2. 解释一下自动装箱和自动拆箱？

#### 先给结论
- 自动装箱：基本类型自动转换成对应的包装类型。
- 自动拆箱：包装类型自动转换成对应的基本类型。
- 这是 Java 5 引入的语法糖，让基本类型和包装类型之间转换更方便。

#### 什么是自动装箱
```java
Integer a = 10;
```

底层可以理解成：
```java
Integer a = Integer.valueOf(10);
```

解释：
- `10` 本来是基本类型 `int`
- 赋值给 `Integer` 时，Java 自动把它转换成包装对象
- 这就是自动装箱

#### 什么是自动拆箱
```java
Integer a = 10;
int b = a;
```

底层可以理解成：
```java
int b = a.intValue();
```

解释：
- `a` 是 `Integer` 对象
- 赋值给 `int` 时，Java 自动取出内部的值
- 这就是自动拆箱

#### 一个最常见的场景：集合
```java
import java.util.ArrayList;
import java.util.List;

public class Demo {
    public static void main(String[] args) {
        List<Integer> list = new ArrayList<>();
        list.add(10);          // 自动装箱
        int x = list.get(0);   // 自动拆箱
        System.out.println(x);
    }
}
```

输出结果：
```java
10
```

#### 自动装箱不等于 `new`
- 这一点很重要。
- `Integer a = 100;` 通常不是 `new Integer(100)`，而更接近 `Integer.valueOf(100)`。
- `valueOf()` 可能会走缓存，所以自动装箱不一定每次都新建对象。

```java
Integer a = 100;
Integer b = 100;

System.out.println(a == b);
```

输出结果：
```java
true
```

解释：
- 这里通常走的是 `Integer.valueOf(100)`。
- `100` 在 `Integer` 默认缓存范围 `-128 ~ 127` 内，所以可能复用同一个对象。

#### 和 `new Integer()` 相关的 3 个高频比较题

### 1）两个 `new Integer()` 比较
```java
Integer a = new Integer(10000);
Integer b = new Integer(10000);

System.out.println(a == b);
System.out.println(a.equals(b));
```

输出结果：
```java
false
true
```

解释：
- `new` 每次都会创建新对象。
- 所以两个 `new Integer()` 的引用一定不同，`==` 为 `false`。
- 但它们包装的数值相同，所以 `equals()` 为 `true`。

### 2）`new Integer()` 和自动装箱 `Integer` 比较
```java
Integer a = new Integer(10000);
Integer b = 10000;

System.out.println(a == b);
System.out.println(a.equals(b));
```

输出结果：
```java
false
true
```

解释：
- `a` 是显式 `new` 出来的堆对象。
- `b` 是自动装箱得到的 `Integer`，底层更接近 `Integer.valueOf(10000)`。
- 两者通常不是同一个对象，所以 `==` 为 `false`。
- 但数值相同，所以 `equals()` 为 `true`。

### 3）两个非 `new` 的 `Integer` 比较
- 自动装箱通常走 `Integer.valueOf()`。
- `Integer.valueOf()` 默认缓存 `-128 ~ 127` 范围内的整数对象。
- 所以两个非 `new` 的 `Integer` 用 `==` 比较时，结果会受缓存影响。

```java
Integer a = 100;
Integer b = 100;
Integer c = 128;
Integer d = 128;

System.out.println(a == b);
System.out.println(c == d);
```

输出结果：
```java
true
false
```

解释：
- `100` 在缓存范围内，通常复用同一个对象。
- `128` 超出默认缓存范围，通常不会复用同一个对象。
- 所以两个非 `new` 的 `Integer` 是否 `==` 为真，要看数值是否落在缓存范围内。

#### 自动拆箱的风险：空指针异常
```java
public class Demo {
    public static void main(String[] args) {
        Integer x = null;
        int y = x; // 自动拆箱
        System.out.println(y);
    }
}
```

运行结果：
```java
Exception in thread "main" java.lang.NullPointerException
```

解释：
- 自动拆箱本质上相当于 `x.intValue()`
- `x` 是 `null`
- 所以会抛出空指针异常

#### 自动装箱和拆箱的优缺点

### 好处
- 代码更简洁
- 基本类型和包装类型之间转换更自然
- 在集合和泛型场景下非常方便

### 风险
- 可能带来空指针异常
- 可能产生性能损耗
- 和 `==` 比较一起出现时容易让人误判

#### 如何从底层理解
- 自动装箱本质：编译器帮你补了 `valueOf()`
- 自动拆箱本质：编译器帮你补了 `xxxValue()`
- 所以它本质上是语法糖，不是 JVM 新增了一套完全不同的机制

#### 面试常见误区
- 自动装箱不等于“总是 `new` 一个对象”
- 两个 `new Integer()` 用 `==` 一定不相等
- 两个非 `new` 的 `Integer` 用 `==` 不一定不相等，可能会受缓存影响
- 自动拆箱不是无风险操作，包装类型为 `null` 时非常容易 NPE

#### 面试回答模板
- 自动装箱是指基本类型自动转换成对应的包装类型，自动拆箱是指包装类型自动转换成对应的基本类型。
- 它是 Java 5 引入的语法糖，本质上分别对应 `valueOf()` 和 `xxxValue()` 这类方法调用。
- 它让集合、泛型等对象场景下写法更简洁，但也可能带来空指针异常、性能损耗以及 `==` 比较误判等问题。

#### 面试简答版
- 自动装箱：基本类型变包装类型。
- 自动拆箱：包装类型变基本类型。
- 自动装箱通常更接近 `valueOf()`，不等于总是 `new`。
- 它方便但有风险，尤其要注意空指针和 `==` 比较问题。
