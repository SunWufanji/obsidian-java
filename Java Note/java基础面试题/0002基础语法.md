# 基础语法

## 1. Java 有哪些数据类型？

#### 先给结论
- Java 数据类型分两大类：基本数据类型和引用数据类型。
- 基本数据类型一共 8 种：`byte`、`short`、`int`、`long`、`float`、`double`、`char`、`boolean`。
- 除了这 8 种之外，像 `String`、数组、类对象、接口实现类对象，本质上都属于引用数据类型。

#### 核心解释
- 基本数据类型保存的是“值”本身，使用起来更直接。
- 引用数据类型保存的是“对象的引用”，真正的数据通常在堆里。
- `char` 表示单个字符，使用单引号，比如 `'A'`。
- `String` 不是基本类型，而是引用类型，使用双引号，比如 `"hello"`。
- `boolean` 只有 `true` 和 `false` 两个值。面试里不要死背它一定占多少字节，因为 Java 规范没有强制规定它的内存大小，具体和 JVM 实现有关。

#### 简单代码示例
```java
public class Demo {
    public static void main(String[] args) {
        int a = 10;
        long b = 10L;
        char c = 'A';
        boolean ok = true;
        String s = "hello";

        System.out.println(a);
        System.out.println(b);
        System.out.println(c);
        System.out.println(ok);
        System.out.println(s);
    }
}
```

#### 输出结果
```text
10
10
A
true
hello
```

#### 输出 / 现象解释
- `10L` 后面的 `L` 表示这是一个 `long` 字面量，不加的话整数字面量默认按 `int` 处理。
- `'A'` 用单引号，因为 `char` 表示单个字符。
- `"hello"` 用双引号，因为它是 `String` 字符串对象。

#### 面试易错点 / 补充点
- `String` 不是基本数据类型，这是很高频的易错点。
- `float` 字面量后面通常要加 `F`，比如 `3.14F`，否则默认会按 `double` 处理。
- `long` 字面量建议写成 `10L`，可读性更好，也避免编译器按 `int` 理解。
- 引用数据类型常见可以先记：类、接口实现对象、数组、`String`。

#### 面试回答模板
- Java 的数据类型分为基本数据类型和引用数据类型两大类。基本数据类型有 8 种，分别是 `byte`、`short`、`int`、`long`、`float`、`double`、`char`、`boolean`。像 `String`、数组、对象这些都属于引用数据类型。基本类型存的是值本身，引用类型存的是对象引用。

#### 面试简答版
- 两类：基本类型、引用类型。
- 基本类型 8 个：`byte`、`short`、`int`、`long`、`float`、`double`、`char`、`boolean`。
- `String` 不是基本类型，是引用类型。

## 2. switch 是否能作用在 byte 上，是否能作用在 long 上，是否能作用在 String 上？

#### 先给结论
- `switch` 可以作用在 `byte` 上。
- `switch` 不可以作用在 `long` 上。
- `switch` 可以作用在 `String` 上，但要求 JDK 7 及以后。

#### 核心解释
- 传统 `switch` 主要支持能转成整型范围比较的类型，比如 `byte`、`short`、`char`、`int`。
- `byte` 虽然本身不是 `int`，但参与 `switch` 判断时会按整型规则处理，所以可以用。
- `long` 不行，这不是语法糖没补上的问题，而是 Java 语言规范本身就不允许。
- `String` 从 JDK 7 开始支持，本质上不是 JVM 直接识别字符串分支，而是编译器帮你转换成基于 `hashCode()` 和 `equals()` 的判断逻辑。

#### 简单代码示例
```java
public class Demo {
    public static void main(String[] args) {
        byte b = 2;
        switch (b) {
            case 1 -> System.out.println("one");
            case 2 -> System.out.println("two");
            default -> System.out.println("other");
        }

        String s = "java";
        switch (s) {
            case "java" -> System.out.println("match java");
            default -> System.out.println("other");
        }
    }
}
```

#### 输出结果
```text
two
match java
```

#### 输出 / 现象解释
- `byte` 分支能正常命中，因为它属于 `switch` 支持的整型兼容范围。
- `String` 分支在编译后会被改写成类似“先比哈希，再做 `equals` 兜底确认”的代码，所以最终也能正常匹配。

#### 运行方式简要解释
##### 1）作用在 `byte` 上时
- 编译器会按整型比较思路处理，底层分支判断走的是整数分支逻辑。

##### 2）作用在 `String` 上时
- 编译器通常会先对字符串求 `hashCode()`，再结合 `equals()` 判断，避免哈希冲突带来误判。

##### 3）为什么不能作用在 `long` 上
- Java 语言规范没开放这个能力。
- 从底层角度看，`switch` 指令体系主要围绕 `int` 设计，不直接支持 `long` 这种 64 位整型分支选择。

#### 面试易错点 / 补充点
- `switch` 还可以作用在 `short`、`char`、`int`、枚举、`String`，以及对应包装类型 `Byte`、`Short`、`Character`、`Integer`。
- `boolean` 也不能用于 `switch`。
- `case` 后面的值必须是编译期常量。
- `String` 能用于 `switch`，但这是从 JDK 7 才开始支持的。

#### 面试回答模板
- `switch` 可以作用在 `byte` 上，不能作用在 `long` 上，可以作用在 `String` 上，但 `String` 是从 JDK 7 开始支持的。`byte` 本质上可以按整型分支处理，而 `String` 的 `switch` 底层一般会编译成 `hashCode + equals` 的判断逻辑。`long` 则是语言规范本身不支持。

#### 面试简答版
- `byte`：可以。
- `long`：不可以。
- `String`：可以，JDK 7+。

## 3. public、private、protected，以及不写（默认）时的区别？

#### 先给结论
- `private`：只能在当前类内部访问。
- 默认不写：只能在同一个包内访问，也叫包访问权限、包级私有。
- `protected`：同包可访问；不同包下，子类也可以访问。
- `public`：任何地方都可以访问。

#### 核心解释
- 这 4 个访问修饰符，本质上控制的是“谁能看到、谁能用”这个成员。
- 权限从小到大可以记成：
  - `private`
  - 默认不写
  - `protected`
  - `public`
- 面试里最容易混的是 `protected`：它不只是“同包可见”，还多了一层“跨包对子类可见”。

#### 访问范围速记表
| 修饰符 | 当前类 | 同包 | 不同包子类 | 不同包非子类 |
| --- | --- | --- | --- | --- |
| `private` | 可以 | 不可以 | 不可以 | 不可以 |
| 默认不写 | 可以 | 可以 | 不可以 | 不可以 |
| `protected` | 可以 | 可以 | 可以 | 不可以 |
| `public` | 可以 | 可以 | 可以 | 可以 |

#### 简单代码示例
```java
class Parent {
    public int a = 1;
    protected int b = 2;
    int c = 3;
    private int d = 4;

    public void show() {
        System.out.println(a + "," + b + "," + c + "," + d);
    }
}

public class Demo {
    public static void main(String[] args) {
        Parent p = new Parent();
        System.out.println(p.a);
        System.out.println(p.b);
        System.out.println(p.c);
        // System.out.println(p.d); // 编译报错，private 不能在类外访问
    }
}
```

#### 输出结果
```text
1
2
3
```

#### 输出 / 现象解释
- `a`、`b`、`c` 都能访问，是因为 `Demo` 和 `Parent` 在同一个包下。
- `d` 访问不了，因为 `private` 只允许在 `Parent` 类内部访问。

#### 面试易错点 / 补充点
- `protected` 在“跨包”时，不是所有地方都能访问，只有子类能访问，而且通常是通过继承关系访问，不是随便拿个父类对象就能点出来。
- 顶层类不能用 `private` 和 `protected` 修饰，顶层类一般只能是 `public` 或默认不写。
- 默认不写不等于 `protected`，默认不写没有“跨包对子类开放”的能力。

#### 面试回答模板
- Java 里访问修饰符有 4 种：`private`、默认不写、`protected`、`public`。`private` 只能当前类访问；默认不写只能同包访问；`protected` 除了同包可访问外，不同包下的子类也可以访问；`public` 则任何地方都可以访问。权限范围从小到大依次是 `private < 默认 < protected < public`。

#### 面试简答版
- `private`：本类。
- 默认：同包。
- `protected`：同包 + 跨包子类。
- `public`：全局可见。

## 4. break、continue、return 的区别及作用？

#### 先给结论
- `break`：结束当前循环或 `switch`。
- `continue`：结束本次循环，直接进入下一次循环。
- `return`：直接结束整个方法；如果方法有返回值，还会把结果一起返回。

#### 核心解释
- `break` 关注的是“跳出当前这层结构”。
- `continue` 关注的是“这次不做了，下一轮继续”。
- `return` 关注的是“方法到此结束，后面的代码都不执行了”。
- 所以它们最大的区别，不只是跳转位置不同，而是“结束的作用域”不同。

#### 简单代码示例
```java
public class Demo {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            if (i == 3) {
                break;
            }
            System.out.print(i + " ");
        }
        System.out.println();

        for (int i = 1; i <= 5; i++) {
            if (i == 3) {
                continue;
            }
            System.out.print(i + " ");
        }
        System.out.println();

        test();
    }

    public static void test() {
        for (int i = 1; i <= 5; i++) {
            if (i == 3) {
                return;
            }
            System.out.print(i + " ");
        }
        System.out.println("end");
    }
}
```

#### 输出结果
```text
1 2
1 2 4 5
1 2
```

#### 输出 / 现象解释
- 第一段在 `i == 3` 时执行 `break`，整个循环直接结束，所以只打印 `1 2`。
- 第二段在 `i == 3` 时执行 `continue`，只是跳过这一次，所以打印 `1 2 4 5`。
- 第三段在 `i == 3` 时执行 `return`，`test()` 方法直接结束，所以后面的 `System.out.println("end");` 根本不会执行。

#### 运行方式简要解释
##### 1）`break`
- 结束当前这一层循环或 `switch`，控制流跳到它后面的第一行代码。

##### 2）`continue`
- 本次循环后面的语句不执行，直接进入下一次循环条件判断。

##### 3）`return`
- 整个方法立刻结束。
- 如果方法定义了返回值，还必须把对应结果一起返回。

#### 面试易错点 / 补充点
- `break` 不等于结束方法，它通常只是结束当前循环或 `switch`。
- `continue` 只能用于循环，不能单独用于普通代码块。
- `return` 不只是“返回值”，即使是 `void` 方法也可以写 `return;`，作用是提前结束方法。
- 多层循环里还有“带标签的 `break` / `continue`”这种写法，但日常开发里用得不算多，知道即可。

#### 面试回答模板
- `break`、`continue`、`return` 都是流程控制语句，但结束范围不同。`break` 是结束当前循环或 `switch`；`continue` 是跳过本次循环，进入下一次循环；`return` 是直接结束整个方法，如果方法有返回值，还要把结果返回出去。

#### 面试简答版
- `break`：跳出当前循环 / `switch`。
- `continue`：跳过本次循环。
- `return`：结束整个方法。
