# Redis 内存回收

## 4.1 过期 Key 处理

Redis 使用两个 Dict 来管理数据：
- `db->dict`：存储所有 key-value 对
- `db->expires`：存储设置了 TTL 的 key 及其过期时间

### 惰性删除

访问某个 key 时，先检查是否过期，过期则删除后返回 nil。

- 优点：不占用额外 CPU
- 缺点：过期 key 如果一直不被访问，内存无法释放

### 周期删除

定时任务周期性抽样部分 key，检查并删除过期的。

**SLOW 模式：**
- 由 `initServer()` 中的定时任务触发，频率由 `server.hz` 控制（默认 10，即每秒 10 次）
- 每次执行不超过 25ms
- 逐个遍历 db 中的 bucket，每次抽取 20 个 key 判断是否过期
- 若过期 key 比例 > 10% 且未超时，继续抽样

**FAST 模式：**
- 由每次事件循环前的 `beforeSleep()` 触发
- 两次 FAST 模式间隔不低于 2ms
- 每次执行不超过 1ms
- 逻辑与 SLOW 相同，但时间限制更严格

**小结：**

| 策略 | 触发时机 | 优点 | 缺点 |
|------|---------|------|------|
| 惰性删除 | 访问 key 时 | 无额外 CPU 开销 | 过期 key 可能长期占用内存 |
| 周期删除 | 定时任务 | 主动释放内存 | 无法保证及时清理所有过期 key |

两种策略配合使用，互为补充。

## 4.2 内存淘汰策略

当 Redis 内存达到 `maxmemory` 上限时，在处理新命令前会触发内存淘汰（`processCommand()` 中执行）。

### 8 种淘汰策略

| 策略 | 淘汰范围 | 淘汰方式 |
|------|---------|---------|
| `noeviction` | — | 不淘汰，内存满时拒绝写入（默认） |
| `volatile-ttl` | 设置了 TTL 的 key | TTL 越小越先淘汰 |
| `volatile-random` | 设置了 TTL 的 key | 随机淘汰 |
| `volatile-lru` | 设置了 TTL 的 key | LRU 算法 |
| `volatile-lfu` | 设置了 TTL 的 key | LFU 算法 |
| `allkeys-random` | 全体 key | 随机淘汰 |
| `allkeys-lru` | 全体 key | LRU 算法 |
| `allkeys-lfu` | 全体 key | LFU 算法 |

### LRU vs LFU

**LRU（Least Recently Used）最近最少使用：**
- 用当前时间减去最后一次访问时间，差值越大优先淘汰
- 存储在 RedisObject 的 `lru` 字段（24 位时钟）

**LFU（Least Frequently Used）最少频率使用：**
- 统计每个 key 的访问频率，频率越低优先淘汰
- 使用**逻辑访问次数**（非精确计数）：
  1. 生成 0~1 之间的随机数 R
  2. 计算 P = 旧次数 × `lfu_log_factor` + 1
  3. 若 R < P，计数器 +1（最大 255）
  4. 访问次数随时间衰减：每隔 `lfu_decay_time` 分钟，计数器 -1

**对比：**

| | LRU | LFU |
|-|-----|-----|
| 关注点 | 最近是否被访问 | 访问频率 |
| 适用场景 | 热点数据有时间局部性 | 热点数据访问频繁但不一定最近 |
| 缺点 | 偶发访问的冷数据可能误判为热数据 | 实现更复杂 |

## 面试简答

**Q: Redis 如何处理过期 key？**

A: 两种策略结合：惰性删除（访问时检查，过期则删除）+ 周期删除（定时任务抽样清理）。惰性删除节省 CPU，周期删除防止内存长期被占用。

**Q: Redis 有哪些内存淘汰策略？**

A: 8 种。按淘汰范围分两类：`allkeys-*` 对全体 key 淘汰，`volatile-*` 只对设置了 TTL 的 key 淘汰。按淘汰方式分：random（随机）、lru（最近最少使用）、lfu（最少频率使用）、ttl（TTL 最小优先）。默认是 `noeviction`，内存满时拒绝写入。

**Q: LRU 和 LFU 的区别？**

A: LRU 关注最近访问时间，长时间未访问的 key 优先淘汰；LFU 关注访问频率，访问次数少的 key 优先淘汰。LFU 能更准确识别真正的冷数据，但实现更复杂。Redis 的 LFU 使用逻辑计数（概率递增 + 时间衰减）来节省内存。
