# 抽象类和接口

## 1. 一句话理解
- 抽象类：更像“半成品父类”或“模板”，回答“它是什么”。
- 接口：更像“能力规范”或“行为合同”，回答“它能做什么”。
- 抽象类偏向代码复用；接口偏向规范约束和解耦扩展。

## 2. 为什么需要两者
- 面向对象里的抽象并不只有一种。
- 一种是对“同一类事物共性”的抽象，比如动物、车辆、员工。
- 另一种是对“某种行为能力”的抽象，比如会飞、可支付、可比较。
- 所以 Java 同时提供了抽象类和接口这两种机制。

## 3. 抽象类是什么
- 抽象类本身不能直接实例化，只能作为父类被继承。
- 它适合沉淀一类对象的公共属性、公共方法和默认实现。
- 抽象类里既可以有抽象方法，也可以有普通方法。
- 抽象类可以有成员变量，也可以有构造器。
- 子类继承抽象类后，需要补全抽象方法，或者自己继续声明为抽象类。

## 4. 接口是什么
- 接口本质上是一种行为规范或能力标准。
- 它主要用来约束实现类必须具备哪些能力，而不强调共享状态。
- 一个类可以实现多个接口，因此接口特别适合做横向能力扩展。
- 接口不能有构造器，也不能保存普通实例状态。
- 接口中的变量本质上都是常量，默认是 `public static final`。
- 现代 Java 中，接口除了抽象方法，还可以有 `default`、`static`，以及 Java 9 引入的 `private` 方法。

## 5. 最核心的区别

### 5.1 设计意图不同
- 抽象类：抽象的是“类的共性”。
- 接口：抽象的是“行为能力”。

### 5.2 关系不同
- 继承抽象类，更像“是不是”的关系。
- 实现接口，更像“有没有这种能力”的关系。

### 5.3 复用方式不同
- 抽象类适合复用属性、构造逻辑和默认实现。
- 接口适合统一调用方式和外部规范。

### 5.4 扩展能力不同
- 一个类只能继承一个抽象类，因为 Java 类是单继承。
- 一个类可以实现多个接口，因此接口更灵活。

## 6. 如何理解“是什么”和“能做什么”
- `Dog`、`Cat` 都是 `Animal`，这是“是什么”，更适合抽象类。
- `Bird`、`Airplane` 都会飞，这是“能做什么”，更适合接口。
- “动物”是一类事物的共性，“飞行”是一种能力，不是一类对象本身。

## 7. 怎么选

### 适合用抽象类的场景
- 多个子类之间关系很强，确实属于同一类事物。
- 子类之间有共享属性、共享代码、共享初始化逻辑。
- 你希望给子类提供一部分默认实现。

### 适合用接口的场景
- 你只想定义一种规范或能力。
- 这种能力可能被很多不相关的类实现。
- 你希望系统解耦，保留更强的扩展性和替换能力。

### 实战判断法
- 需要“状态 + 默认实现 + 强共性” -> 优先抽象类。
- 需要“规范 + 解耦 + 多实现” -> 优先接口。

## 8. 开发中常见的组合方式
- 接口负责定义规范。
- 抽象类负责沉淀公共实现。
- 具体类继承抽象类并实现接口。

### 典型理解
- 接口负责说“你必须会做什么”。
- 抽象类负责说“这些公共东西我先帮你做好一部分”。
- 具体类负责“把最后差异化的部分补上”。

## 9. 容易混淆的点
- 抽象类不一定非要有抽象方法，只要类被 `abstract` 修饰，它就是抽象类。
- 有抽象方法的类必须是抽象类。
- 接口不再是“只能有抽象方法”的旧模型，现代 Java 中还能有 `default`、`static`、`private` 方法。
- 抽象类可以有构造器，因为子类对象中包含父类那部分状态，需要父类构造逻辑完成初始化。
- 接口不能有构造器，因为接口不是对象模板，而是能力规范。
- 抽象类不能用 `final` 修饰，抽象方法也不能用 `final` 修饰，因为这两组语义冲突。

## 10. 面试回答模板
- 抽象类和接口都用于抽象和实现多态，但设计目的不同。
- 抽象类更适合抽取一类事物的共性，可以包含成员变量、构造器、普通方法和抽象方法，偏向代码复用和模板设计。
- 接口更适合定义行为规范和能力标准，一个类可以实现多个接口，偏向解耦和扩展。
- 简单来说，抽象类表示“是什么”，接口表示“能做什么”；需要共享状态和默认实现时用抽象类，需要定义规范和支持多实现时用接口。

## 11. 速记版
- 抽象类：同宗同源，抽共性，带状态，能复用。
- 接口：不论出身，定规范，重能力，可多实现。
