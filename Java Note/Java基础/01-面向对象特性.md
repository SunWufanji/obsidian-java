## 1.继承
#### 1.1 继承的概念和实例
- **概念**：继承就是子类继承父类的特征和行为，使得子类对象（实例）具有父类的实例域和方法，或子类从父类继承方法，使得子类具有父类相同的行为。
- **继承实例**：

> [!example]- 点击展开/收起代码
> ```java
> //公共父类
>   public class Animal { 
>     private String name;  
>     private int id; 
>     public Animal(String myName, int myid) { 
>         name = myName; 
>         id = myid;
>     } 
>     public void eat(){ 
>         System.out.println(name+"正在吃"); 
>     }
>     public void sleep(){
>         System.out.println(name+"正在睡");
>     }
>     public void introduction() { 
>         System.out.println("大家好！我是"         + id + "号" + name + "."); 
>     } 
> }
> //企鹅类
>   public class Penguin extends Animal { 
>     public Penguin(String myName, int myid) { 
>         super(myName, myid); 
>     } 
> }
> //老鼠类
> public class Mouse extends Animal { 
>     public Mouse(String myName, int myid) { 
>         super(myName, myid); 
>     } 
> }
> ```

- **extends关键字**：在 Java 中，类的继承是==单一继承==，也就是说，一个子类只能拥有一个父类，所以 extends 只能继承一个类。
- **implements关键字**：使用 implements 关键字可以变相的使java具有==多继承==的特性，使用范围为类继承接口的情况，可以同时继承多个接口（接口跟接口之间采用逗号分隔）。

> [!example]- 点击展开/收起接口实例
> ```java
> public interface A {
>     public void eat();
>     public void sleep();
> }
>  
> public interface B {
>     public void show();
> }
>  //多继承（接口）
> public class C implements A,B {
> }
> ```

- **super 关键字**: 我们可以通过 super 关键字来实现对父类成员的访问，用来引用当前对象的父类。
- **this关键字**: 指向自己的引用，引用当前对象，即它所在的方法或构造函数所属的对象实例。

> [!tips]
> this和super关键字只能出现在非static修饰的代码中。

- **实例**：

> [!example]- 点击展开/收起 super/this 测试代码
> ```java
> class Animal {
>     void eat() {
>         System.out.println("animal : eat");
>     }
> }
>  
> class Dog extends Animal {
>     void eat() {
>         System.out.println("dog : eat");
>     }
>     void eatTest() {
>         this.eat();   // this 调用自己的方法
>         super.eat();  // super 调用父类方法
>     }
> }
>  
> public class Test {
>     public static void main(String[] args) {
>         Animal a = new Animal();
>         a.eat();
>         Dog d = new Dog();
>         d.eatTest();
>     }
> }
> ```

> [!success]- 输出结果
> animal : eat
> dog : eat
> animal : eat

- **final 关键字**: 使用 final 关键字声明类，就是把类定义定义为最终类，不能被继承，或者用于修饰方法，该方法不能被子类重写。
#### 1.2 实现多继承的方式
- **内部类**可以继承一个与外部类无关的类，保证了内部类的独立性，正是基于这一点，可以达到多继承的效果。
- **多层继承**：子类继承父类，父类如果还继承其他的类，那么这就叫**多层继承**。这样子类就会拥有所有被继承类的属性和方法。

#### 1.3 重写与重载
- **重写(Override)**：**外壳不变，但是核心内容重写**。
- **重载(Overload)**：**方法名相同，但参数不一致，那么可以说一个方法是另一个方法的重载。(必须改变参数列表)**

|  区别点  |           方法重写            |      方法重载      |
| :---: | :-----------------------: | :------------: |
|  结构上  |     垂直结构，是一种父子类之间的关系      | 水平结构，是一种同类之间关系 |
| 参数列表  |           不可以修改           |      可以修改      |
| 访问修饰符 | 子类的访问修饰符范围必须大于等于父类访问修饰符范围 |      可以修改      |
| 抛出异常  | 子类方法异常必须是父类方法异常或父类方法异常子异常 |      可以修改      |

#### 1.4 修饰符
##### 1.4.1访问修饰符
**`public，protected，default`(无修饰词)，`private`修饰符是面向对象中非常重要的知识点，而在继承中也需要懂得各种修饰符使用规则。
- `private`:访问权限最窄，私有的，只能被本类对象访问。
- `default`（或`friendly`):默认的，只允许在一个包中访问。
- `protected`:只能被类本身的方法及子类访问,即使子类在不同的包中也可以访问。(比 default 宽一点)
- `public`:**最宽**的修饰符,可以跨包跨类。

> [!tip]
> **子类继承父类的访问修饰符作用域不能比父类小，假如父类是protected修饰的，其子类只能是protected或者public，绝对不能是default(默认的访问范围)或者private。所以在继承中需要重写的方法不能使用private修饰词修饰。(异常也是如此）**

##### 1.4.2非访问修饰符：
###### **`static` 修饰符**: 
==是静态变量，静态方法==，如果在一个类中使用static修饰变量或者方法的话，它们**可以直接通过类访问，不需要创建一个类的对象来访问成员。**
**规则**：
- 构造方法不允许声明为 `static`。
- 静态方法中不存在当前对象，因而不能使用 this，当然也不能使用 super。
- 静态方法不能被非静态方法重写(覆盖)
- 静态方法能被静态方法重写(覆盖)
###### **`final`修饰符**:
**`final`变量**：变量一旦赋值后，不能被重新赋值。
**`final`方法**：**父类中的 final 方法可以被子类继承，但是不能被子类重写**。
**`final`  类** ：**final 类不能被继承**。
###### **`abstract` 修饰符**:
- abstract 英文名为“抽象的”，主要用来修饰类和方法，称为抽象类和抽象方法。
- **抽象方法**：有很多不同类的方法是相似的，但是具体内容又不太一样，所以我们只能抽取他的声明，没有具体的方法体，即抽象方法可以表达概念但无法具体实现。
- **抽象类**：**有抽象方法的类必须是抽象类**，抽象类可以表达概念但是无法构造实体的类。

#### 1.5 `Object`类和转型
##### 1.5.1`Object`类概述
1. `Object`是类层次结构的**根类**，所有的类都隐式的继承自Object类。
2. `Java`所有的对象都拥有`Object`默认方法
3. `Object`类的构造方法有一个，并且是**无参构造**
4. 如果一个类没有显式声明它的父类,那么默认这个类的父类就是`Object`类，任何类都可以使用`Object`类的方法,也可和`Object`进行向上、向下转型
- **`toString()`**方法表示返回该对象的字符串，由于各个对象构造不同所以需要重写，如果不重写的话默认返回`类名@hashCode`格式。
##### 1.5.2  向上转型
- **向上转型**：==把儿子当父亲来看==，通过子类对象(小范围)实例化父类对象(大范围)，这种属于自动转换。
- ![[Pasted image 20260414220437.png|554]]

##### 1.5.3  向下转型
- **向下转型** :==把名义上的父亲，变回真实的儿子身份==，即通过父类对象(大范围)实例化子类对象(小范围)
- **规则**：**先经过向上转型，才能向下转型**
- 向上转型后儿子特有的技能就没了，为了这些技能得转回去

#### 1.6 继承中子父类初始化顺序
1. 父类中静态成员变量和静态代码块
2. 子类中静态成员变量和静态代码块
3. 父类中普通成员变量和代码块，父类的构造函数
4. 子类中普通成员变量和代码块，子类的构造函数
5. **静态>非静态，父类>子类，非构造函数>构造函数**
## 2.多态
#### 2.1概念及实例
 多态就是同一个接口，使用不同的实例而执行不同操作。
###### **多态存在的三个必要条件**：
- 继承
- 重写
- **向上转型（Upcasting）**：父类引用指向子类对象。 
###### 多态的实现方式：
- 重写：override以及overload（重载）
- 接口：具体见java接口章节
- 抽象类和抽象方法：具体见[[03-抽象类和接口]]
> [!example]- 代码实例
> ```Test.java
> public class Test {
>     public static void main(String[] args) {
>       show(new Cat());  // 以 Cat 对象调用 show 方法
>       show(new Dog());  // 以 Dog 对象调用 show 方法
>                 
>       Animal a = new Cat();  // 向上转型  
>       a.eat();               // 调用的是 Cat 的 eat
>       Cat c = (Cat)a;        // 向下转型  
>       c.work();        // 调用的是 Cat 的 work
>   }  
>             
>     public static void show(Animal a)  {
>       a.eat();  
>         // 类型判断
>         if (a instanceof Cat)  {  // 猫做的事情 
>             Cat c = (Cat)a;  
>             c.work();  
>         } else if (a instanceof Dog) { // 狗做的事情 
>             Dog c = (Dog)a;  
>             c.work();  
>         }  
>     }  
> }
>  
> abstract class Animal {  
>     abstract void eat();  
> }  
>   
> class Cat extends Animal {  
>     public void eat() {  
>         System.out.println("吃鱼");  
>     }  
>     public void work() {  
>         System.out.println("抓老鼠");  
>     }  
> }  
>   
> class Dog extends Animal {  
>     public void eat() {  
>         System.out.println("吃骨头");  
>     }  
>     public void work() {  
>         System.out.println("看家");  
>     }  
> }
> ```

> [!success]- 输出结果
> 吃鱼
> 抓老鼠
> 吃骨头
> 看家
> 吃鱼
> 抓老鼠

## 4.Question
### 1.成员变量与局部变量的区别有哪些？
- **语法层面（定义位置）**：
    
    - **成员变量**：定义在类中、方法外。
        
    - **局部变量**：定义在方法内或方法参数中。
        
- **存储层面（内存位置）**：
    
    - **成员变量**：随对象一起存在**堆内存**中。
        
    - **局部变量**：存在于**栈内存**的方法栈帧中。
        
- **生命周期**：
    
    - **成员变量**：随对象创建而出生，随对象被回收而消失。
        
    - **局部变量**：随方法调用而出生，随方法结束而销毁。
        
- **默认值**：
    
    - **成员变量**：有默认值（如 int 为 0，引用类型为 null）。
        
    - **局部变量**：**没有默认值**，必须先赋值（初始化）才能使用，否则报错。

### 2. 构造方法有哪些特性？

- **名字**：必须与类名完全相同。
    
- **返回值**：**没有返回值类型**，甚至不能写 void。
    
- **触发时机**：在使用 new 关键字创建对象时自动调用，且仅调用一次。
    
- **默认性**：如果你没写构造方法，系统会默认送你一个无参构造；但如果你写了任何一个构造方法，系统就不再赠送。
    
- **重载**：一个类可以有多个构造方法（参数个数或类型不同）。
    

---

### 3. 在一个静态方法内调用一个非静态成员为什么是非法的？

- **核心逻辑**：**先有类，后有对象。**
    
- **静态（static）方法**是属于**类**的，它在类加载时就存在了。
    
- **非静态成员**是属于**对象**的，只有 new 了一个对象后它才存在。
    
- **结论**：静态方法运行的时候，对象可能根本还没创建。既然没有对象，也就没有“非静态成员”，所以是非法的。这就好比“你还没出生，就想花你儿子的钱”是不可能的。

### 4. 面向对象的三大特性？

1. **封装（Encapsulation）**：隐藏内部实现细节（属性 private），通过公共方法（Getter/Setter）暴露功能。
    
2. **继承（Inheritance）**：子类继承父类的属性和方法（extends），实现代码复用。
    
3. **多态（Polymorphism）**：父类引用指向子类对象。同一个接口，根据对象的不同实现，表现出不同的行为。
    

---

### 5. 从 JVM 的角度解释一下多态？

- **动态绑定（Dynamic Binding）**：
    
    - 在编译期，JVM 只知道引用变量的类型（左边）。
        
    - 但在运行期，当执行方法调用时，JVM 会使用指令（如 invokevirtual）去**堆内存**中寻找该引用实际指向的对象。
        
    - 它会查看该对象的“虚方法表（vtable）”，找到实际重写（Override）后的方法代码地址并执行。
        
- **结论**：多态的本质是 JVM 在**运行时刻**才确定到底调用哪个类的方法。

### 6. 重载（Overload）和重写（Override）的区别是什么？

|   |   |   |
|---|---|---|
|区别点|重载 (Overload)|重写 (Override)|
|**范围**|同一个类中|子类与父类之间|
|**方法名**|必须相同|必须相同|
|**参数列表**|**必须不同** (个数、类型或顺序)|**必须相同**|
|**返回类型**|可以不同|必须相同（或其子类）|
|**多态类型**|编译时多态（静态绑定）|运行时多态（动态绑定）|

### 7. Java 是否支持多继承？

- **类（Class）层面**：**不支持**。一个类只能 extends 一个父类。这是为了避免“菱形继承”带来的冲突（比如两个爸爸都有同名方法，儿子不知道听谁的）。
    
- **接口（Interface）层面**：**支持**。
    
    - 一个类可以实现（implements）多个接口（如你写的 Dancer implements IDance, IFight）。
        
    - 一个接口也可以继承（extends）多个接口。
