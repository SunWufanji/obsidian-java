# 构造器

## 1. 什么是构造器
- 构造器（Constructor）是对象创建时自动调用的一种特殊方法，主要作用是完成对象初始化。
- `new` 负责创建对象，构造器负责给对象赋初值、保证对象一创建出来就处于可用状态。
- 构造器名必须和类名完全相同。
- 构造器没有返回值类型，连 `void` 都不能写。

```java
class Person {
    String name;
    int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

```java
Person p = new Person("Tom", 18);
```

- 上面代码中，`new Person("Tom", 18)` 创建对象时会自动调用构造器完成初始化。

## 2. 构造器和普通方法的区别

| 对比点 | 构造器 | 普通方法 |
|---|---|---|
| 方法名 | 必须和类名相同 | 自定义 |
| 返回值类型 | 没有，不能写 `void` | 必须有，或写 `void` |
| 调用时机 | 创建对象时自动调用 | 需要手动调用 |
| 主要作用 | 初始化对象 | 完成功能逻辑 |

## 3. 构造器的特点
- 构造器可以重载，只要参数列表不同即可。
- 构造器不能被继承。
- 构造器不能被重写。
- 构造器可以有访问修饰符，比如 `public`、`protected`、默认、`private`。
- 构造器不能用 `static`、`final`、`abstract` 修饰。

## 4. 默认无参构造器
- 如果一个类中没有写任何构造器，编译器会默认生成一个无参构造器。

```java
class A {
}
```

- 上面代码编译后效果相当于：

```java
class A {
    public A() {
    }
}
```

- 但只要你手写了任意一个构造器，编译器就不会再赠送默认无参构造器。

```java
class A {
    public A(String name) {
    }
}
```

- 这时如果再写 `new A()`，就会编译报错。

## 5. 构造器重载
- 同一个类中可以有多个构造器，这叫构造器重载。
- 本质和普通方法重载一样，要求参数列表不同。

```java
class User {
    String name;
    int age;

    public User() {
    }

    public User(String name) {
        this.name = name;
    }

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

## 6. `this()` 和 `super()` 在构造器中的用法

### 6.1 `this()`
- `this()` 用来调用本类的其他构造器。
- 常用于复用本类构造逻辑，避免重复代码。

```java
class Person {
    String name;
    int age;

    public Person() {
        this("Tom", 18);
    }

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

### 6.2 `super()`
- `super()` 用来调用父类构造器。
- 子类创建对象时，必须先初始化父类部分，再初始化自己。

```java
class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }
}

class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }
}
```

### 6.3 高频规则
- `this()` 和 `super()` 都必须写在构造器第一行。
- `this()` 和 `super()` 不能同时出现。
- 如果子类构造器中没有显式写 `super(...)`，编译器会默认加一个 `super()`。
- 如果父类没有无参构造器，而子类又没有显式调用父类有参构造器，就会编译报错。

## 7. 为什么构造器不能被继承和重写
- 构造器名字必须和类名相同。
- 父类和子类类名不同，所以子类不可能“继承”到一个和自己类名相同的构造器。
- 重写的前提是子类继承父类方法后再重新实现，而构造器本身不被继承，所以也不能重写。
- 子类只能通过 `super(...)` 调用父类构造器，这不属于重写。

## 8. 构造器执行顺序

### 8.1 单个类创建对象时
- 成员变量先进行默认初始化。
- 然后执行显式赋值和实例代码块。
- 最后执行构造器。

### 8.2 继承场景下创建对象时
- 父类静态变量、静态代码块
- 子类静态变量、静态代码块
- 父类实例变量、实例代码块
- 父类构造器
- 子类实例变量、实例代码块
- 子类构造器

### 8.3 记忆口诀
- 先静态，后非静态
- 先父类，后子类
- 先成员/代码块，后构造器

## 9. `private` 构造器有什么用
- `private` 构造器可以禁止外部直接创建对象。
- 常见用途有：
  - 单例模式
  - 工具类禁止实例化

```java
class Utils {
    private Utils() {
    }
}
```

- 这样外部就不能写 `new Utils()`。

## 10. 高频面试题

### 10.1 构造器有返回值吗
- 没有。
- 连 `void` 都不能写。

### 10.2 构造器可以重载吗
- 可以。
- 参数列表不同即可。

### 10.3 构造器可以重写吗
- 不可以。
- 因为构造器不会被继承。

### 10.4 构造器可以被继承吗
- 不可以。

### 10.5 构造器可以是 `static` 吗
- 不可以。
- 构造器属于对象初始化逻辑，`static` 属于类级别，两者冲突。

### 10.6 构造器可以是 `final` 吗
- 不可以。
- `final` 的意义主要是防止重写，而构造器本来就不能重写。

### 10.7 构造器可以是 `abstract` 吗
- 不可以。
- `abstract` 表示没有实现，要靠子类重写；而构造器不能重写。

### 10.8 子类构造器一定会调用父类构造器吗
- 会。
- 显式写 `super(...)` 或编译器默认补 `super()`。

## 11. 面试回答模板
- 构造器是对象创建时自动调用的特殊方法，主要用于初始化对象。
- 它的特点是方法名和类名相同，没有返回值类型，创建对象时由 `new` 自动触发。
- 构造器可以重载，但不能被继承，也不能被重写。
- 如果类中没有定义任何构造器，编译器会默认生成一个无参构造器；但只要手写了任意构造器，默认无参构造器就不会再生成。
- 在继承中，子类构造器会通过 `super(...)` 调用父类构造器，`this()` 和 `super()` 都必须放在第一行，这也是常见考点。

## 12. 面试速记版
- 构造器：对象初始化方法。
- 特点：同名、无返回值、自动调用。
- 可重载，不可继承，不可重写。
- 不写任何构造器时，编译器才会赠送默认无参构造器。
- 子类构造器会先调父类构造器。
- `this()` / `super()` 必须放第一行。
