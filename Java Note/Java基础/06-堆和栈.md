# Java 堆和栈

## 1. 什么是堆和栈？

### 核心理解
- 栈（Stack）主要存放方法调用相关的信息，比如局部变量、参数、方法出口等。
- 堆（Heap）主要存放对象实例和数组。
- 你可以简单记成：`局部的、临时的 -> 栈`，`new 出来的对象 -> 堆`。

### 一句话总结
- 栈管“方法运行”。
- 堆管“对象存活”。

## 2. 栈里放什么？
- 方法参数
- 局部变量
- 基本类型的局部变量值
- 引用类型变量本身（引用地址值）
- 方法调用形成的栈帧

### 例子
```java
public class Demo {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;
        Student s = new Student("Tom");
    }
}
```

这里可以这样理解：
- `a`、`b` 这两个局部变量的值在栈中。
- `s` 这个引用变量本身在栈中。
- `new Student("Tom")` 创建出来的对象在堆中。

## 3. 堆里放什么？
- 对象实例
- 数组对象
- 对象中的成员变量

### 例子
```java
class Student {
    String name;

    Student(String name) {
        this.name = name;
    }
}

public class Demo {
    public static void main(String[] args) {
        Student s = new Student("Tom");
    }
}
```

可以这样理解：
- `s` 在栈里，保存的是对象地址值。
- `Student` 对象本体在堆里。
- `name` 作为对象成员，跟着 `Student` 对象一起在堆里。

## 4. 一个经典图景怎么理解？

```java
public class Demo {
    public static void main(String[] args) {
        int x = 100;
        Person p = new Person();
        p.name = "Alice";
    }
}
```

可以脑补成：
- 栈中有两个变量：`x` 和 `p`
- `x` 直接存 `100`
- `p` 存的是某个对象地址
- 堆中有一个 `Person` 对象
- 这个 `Person` 对象里有成员变量 `name`

## 5. 堆和栈的主要区别

| 对比项 | 栈 | 堆 |
|---|---|---|
| 存放内容 | 局部变量、方法调用信息 | 对象、数组、实例数据 |
| 生命周期 | 方法结束就释放 | 对象没有引用后，等待 GC 回收 |
| 管理方式 | 自动、速度快 | 由 JVM 垃圾回收管理 |
| 是否线程共享 | 线程私有 | 线程共享 |
| 常见异常 | `StackOverflowError` | `OutOfMemoryError` |

## 6. 为什么说“引用在栈，对象在堆”？
这句话是 Java 初学阶段最常见的总结。

例如：
```java
Person p = new Person();
```

这行代码拆开理解：
- `p` 是一个局部引用变量，在栈中。
- `new Person()` 创建的对象在堆中。
- `p` 里存的是这个堆对象的地址值。

所以：
- 变量 `p` 自己在栈里
- `p` 指向的对象在堆里

## 7. 方法调用为什么和栈有关？
因为 Java 每调用一个方法，JVM 都会为这个方法创建一个栈帧，压入虚拟机栈。

方法执行结束后：
- 这个栈帧会出栈
- 对应的局部变量也随之销毁

### 例子
```java
public class Demo {
    public static void main(String[] args) {
        test();
        System.out.println("main end");
    }

    public static void test() {
        int x = 10;
        System.out.println("test");
    }
}
```

输出结果：
```java
test
main end
```

执行过程：
- `main()` 入栈
- 调用 `test()` 时，`test()` 对应的栈帧再入栈
- `test()` 执行完出栈
- 然后继续执行 `main()`

## 8. 为什么递归太深会栈溢出？
因为每调用一次方法，都会压入一个新的栈帧。

如果递归没有结束条件，或者递归层数太深：
- 栈空间会被不断占用
- 最终抛出 `StackOverflowError`

### 例子
```java
public class Demo {
    public static void test() {
        test();
    }

    public static void main(String[] args) {
        test();
    }
}
```

运行结果：
```java
Exception in thread "main" java.lang.StackOverflowError
```

## 9. 为什么对象太多会堆内存溢出？
因为对象都放在堆里。

如果：
- 不断 `new` 对象
- 对象又一直不能被回收
- 堆空间被占满

就可能出现 `OutOfMemoryError`

### 示例（理解即可）
```java
import java.util.ArrayList;
import java.util.List;

public class Demo {
    public static void main(String[] args) {
        List<byte[]> list = new ArrayList<>();
        while (true) {
            list.add(new byte[1024 * 1024]);
        }
    }
}
```

可能出现：
```java
Exception in thread "main" java.lang.OutOfMemoryError: Java heap space
```

## 10. 还有哪些内存区域容易一起考？
面试里有时不会只问堆和栈，还会顺带问：
- 方法区 / 元空间
- 程序计数器
- 本地方法栈

你基础阶段先这样记就够了：
- 栈：方法运行
- 堆：对象存储
- 方法区 / 元空间：类相关信息、常量、方法元数据等

补充说明：
- 从 JVM 规范角度叫“方法区”。
- HotSpot 里 JDK 8 以后常说“元空间（Metaspace）”。

## 11. 面试常见误区
- 误区一：基本类型一定都在栈里。  
  不严谨。更准确说法是：**局部基本类型变量通常在栈帧中**；如果是对象成员变量，那它跟着对象在堆里。

- 误区二：引用类型都在堆里。  
  不严谨。更准确说法是：**引用变量本身通常在栈里，引用指向的对象在堆里**。

- 误区三：栈比堆“高级”。  
  不是。它们只是职责不同，一个负责方法执行，一个负责对象存储。

## 12. 面试回答模板
- Java 中，栈主要存放方法调用相关的信息，比如局部变量、参数和栈帧；堆主要存放对象实例和数组。
- 方法每调用一次，就会入栈一个新的栈帧，方法结束后出栈，所以栈的生命周期通常跟方法一致。
- 通过 `new` 创建出来的对象通常存放在堆中，对象没有引用后会由垃圾回收器回收。
- 栈是线程私有的，堆是线程共享的；栈空间过深可能导致 `StackOverflowError`，堆空间不足可能导致 `OutOfMemoryError`。

## 13. 面试简答版
- 栈存方法调用和局部变量，堆存对象和数组。
- 局部变量中的基本类型值通常在栈里；引用变量在栈里，对象本身在堆里。
- 方法太深会栈溢出，对象太多会堆溢出。
