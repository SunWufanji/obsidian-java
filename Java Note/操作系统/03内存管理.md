# 内存管理

## 1. 什么是分页？

### 先说场景

程序运行时需要用内存，但内存不是一整块永远空着等某个程序来用。

如果要求一个程序必须连续放在内存里，就会很麻烦：前面空一点、后面空一点，看起来总空闲不少，但就是凑不出一大块连续空间。

分页就是为了解决这个问题：**把程序和内存都切成一样大的小块，程序不用连续放，只要能通过一张表找到每一页实际放在哪就行。**

### 大白话解释

可以把分页想象成寄快递。

一个大包裹太占地方，仓库不一定能腾出连续的一大片位置，于是把它拆成几个大小差不多的小箱子，分别放到仓库不同格子里。

但是拆开之后，必须有一张清单记录：

```text
第 0 个小箱子 -> 仓库第 2 格
第 1 个小箱子 -> 仓库第 3 格
第 2 个小箱子 -> 仓库第 5 格
```

这张清单，在操作系统里就叫 **页表**。

### 对应到操作系统

| 概念 | 大白话理解 |
|------|------------|
| 页 Page | 程序被切出来的小块 |
| 页框 Frame | 物理内存被切出来的小块 |
| 页表 Page Table | 记录“第几页放在哪个页框”的映射表 |
| 页内偏移 | 进入某一页之后，具体找第几个字节 |

程序看到的是自己的逻辑地址，好像内存从 0 开始连续排着。

但真实内存里，它的第 0 页、第 1 页、第 2 页，可能分别放在物理内存的第 2 块、第 3 块、第 5 块。

### 地址是怎么找到的？

分页系统里，一个逻辑地址通常可以拆成两部分：

```text
逻辑地址 = 页号 + 页内偏移
```

访问内存时大概分两步：

```text
1. 根据页号查页表，找到对应的物理块号
2. 用 物理块号 + 页内偏移，拼出真正的物理地址
```

比如：

```text
页号 2 -> 页表里查到它在物理块 5
页内偏移是 100

最终访问的位置就是：物理块 5 里的第 100 个字节
```

### 为什么访问分页内存可能更慢？

因为它不是直接去内存里拿数据，而是要先查页表。

也就是说，最朴素的分页访问可能需要两次内存访问：

```text
第一次：访问页表，找到物理块号
第二次：访问真正的数据
```

这就像你去仓库拿东西，不能直接冲过去拿，得先看清单，再去对应格子找。

所以 CPU 里通常会加一个 **TLB**，也叫快表，可以理解成“页表的缓存”。

最近查过的页号和物理块号会放到 TLB 里，下次再访问同一页时，就不用每次都去内存里查页表了。

### 分页解决了什么问题？

分页最大的好处是：**程序不用连续存放在物理内存里。**

这样可以减少外部碎片，提高内存利用率。

| 问题 | 分页怎么解决 |
|------|--------------|
| 程序必须连续存放太麻烦 | 分页后每一页可以放在不同物理块里 |
| 外部碎片严重 | 页和页框大小固定，方便分配 |
| 地址转换复杂 | 通过页表完成逻辑地址到物理地址的映射 |

### ⚠️ 面试易错点

- **页是逻辑概念，页框是物理概念**：程序被切成“页”，内存被切成“页框”。
- **页和页框大小相同**：这样页才能刚好放进页框里。
- **分页解决的是外部碎片，不是内部碎片**：最后一页可能装不满，所以仍然可能有内部碎片。
- **页表一般放在内存中**：所以单纯分页访问可能要先查页表，再访问数据。
- **TLB 不是页表本身**：TLB 是页表的高速缓存，用来减少查页表的开销。

### 面试回答模板

分页是操作系统的一种内存管理方式。它把程序的逻辑地址空间划分成大小相等的页，也把物理内存划分成同样大小的页框，然后通过页表记录“页号到页框号”的映射关系。

这样程序就不需要连续存放在物理内存中，可以减少外部碎片，提高内存利用率。访问地址时，CPU 会根据逻辑地址中的页号查页表，找到物理块号，再结合页内偏移得到真正的物理地址。

不过分页也会带来查页表的开销，所以实际系统里通常会使用 TLB，也就是快表，来缓存最近使用过的页表项，加快地址转换。

### 面试简答版

分页就是把程序和物理内存都切成固定大小的小块，程序里的块叫页，内存里的块叫页框，再用页表记录每一页实际放在哪个页框里。

它的好处是程序不用连续存放，能减少外部碎片；缺点是访问内存时需要查页表，所以通常会用 TLB 来加速。

## 2. 什么是分段？

### 先说场景

分页主要是操作系统为了方便管理内存，把程序和内存都切成固定大小的小块。

但程序员写程序时，脑子里想的不是“第 0 页、第 1 页”，而是：

- 这块是代码
- 这块是全局变量
- 这块是堆
- 这块是栈
- 这块是共享库

这些部分的作用不一样，大小也不一样。分段就是为了更贴近程序本身的逻辑结构。

一句话说：**分段是把程序按逻辑功能分成多个段，每个段可以有自己的长度和权限。**

### 大白话解释

可以把一个程序想象成一本书。

分页像是把书硬生生按每 10 页切一刀，不管这一刀是不是刚好切在一章中间。

分段更像是按章节来分：

```text
第 0 段：正文
第 1 段：目录
第 2 段：附录
第 3 段：索引
```

每一章长度可以不同，而且每一章都有自己的起始位置。

操作系统要做的事情，就是用一张表记录：

```text
第 0 段 -> 从物理内存 40KB 开始，长度 30KB
第 1 段 -> 从物理内存 80KB 开始，长度 20KB
第 2 段 -> 从物理内存 120KB 开始，长度 15KB
```

这张表就叫 **段表**。

### 对应到操作系统

| 概念 | 大白话理解 |
|------|------------|
| 段 Segment | 程序里一个有独立意义的逻辑部分 |
| 段号 | 要访问第几个段 |
| 段内地址 | 进入这个段之后，再偏移多少 |
| 段表 | 记录每个段的起始地址和段长 |
| 基址 | 这个段在物理内存里的起点 |
| 段长 | 这个段最多能访问多大范围 |

分段系统里的逻辑地址通常是二维的：

```text
逻辑地址 = 段号 + 段内地址
```

比如：

```text
段号 = 1
段内地址 = 5KB
```

意思是：我要访问第 1 段里面，偏移 5KB 的位置。

### 地址是怎么转换的？

分段访问内存时，大概分三步：

```text
1. 根据段号查段表
2. 检查段内地址有没有超过段长
3. 用 段基址 + 段内地址 得到物理地址
```

比如段表里记录：

```text
第 1 段：基址 80KB，段长 20KB
```

现在访问：

```text
段号 1，段内地址 5KB
```

因为 5KB 没有超过段长 20KB，所以地址合法。

真正的物理地址就是：

```text
80KB + 5KB = 85KB
```

如果访问的是：

```text
段号 1，段内地址 25KB
```

那就越界了，因为第 1 段只有 20KB 长。

### 分段有什么好处？

分段的核心好处是：**符合程序的逻辑结构。**

代码段、数据段、栈段本来就是不同用途，分开管理会更自然。

| 好处 | 怎么理解 |
|------|----------|
| 方便共享 | 多个进程可以共享同一段代码 |
| 方便保护 | 代码段可以只读，数据段可以读写 |
| 符合程序结构 | 程序本来就有代码、数据、栈等逻辑部分 |
| 支持动态增长 | 栈段、堆段可以按需要变化 |

比如代码段通常不应该被修改，所以可以设置成只读。

这就像图书馆里的参考书，只能看，不能在上面乱写。

### 分段的问题

分段也有缺点：**容易产生外部碎片。**

因为每个段长度不一样，段和段之间又是离散分配的，用久了以后，内存里可能留下很多大小不一的小空洞。

这些小空洞单独看都有空间，但不一定能放下一个新的大段。

这点和分页不一样：分页因为块大小固定，所以更方便分配。

### ⚠️ 面试易错点

- **分页是固定大小，分段是长度可变**：页大小一样，段大小不一定一样。
- **分页偏操作系统管理，分段偏程序逻辑结构**：分页是为了提高内存利用率，分段是为了更符合代码、数据、栈这些逻辑需求。
- **分段会产生外部碎片**：因为每个段大小不固定，空闲空间可能被切得七零八落。
- **段表里记录的是基址和段长**：访问时要先判断段内地址是否越界，再加上基址得到物理地址。
- **分段地址是二维的**：段号 + 段内地址，不是单纯一个连续地址。

### 面试回答模板

分段是一种内存管理方式，它按照程序的逻辑结构把程序划分成多个段，比如代码段、数据段、栈段等。每个段的长度可以不同，并且每个段在内存中可以离散存放。

在分段系统中，逻辑地址由段号和段内地址组成。CPU 根据段号查段表，段表中保存了段的基址和段长。访问时先判断段内地址是否超过段长，如果没有越界，就用段基址加段内地址得到真正的物理地址。

分段的优点是符合程序的逻辑结构，方便共享和保护，比如代码段可以设置为只读，也可以被多个进程共享。缺点是段的长度不固定，容易产生外部碎片。

### 面试简答版

分段就是按照程序的逻辑功能把内存划分成多个段，比如代码段、数据段、栈段。逻辑地址由段号和段内地址组成，通过段表找到段的基址和段长，再计算出物理地址。

它的好处是符合程序结构，方便共享和保护；缺点是段大小不固定，容易产生外部碎片。

## 3. 分页和分段有什么区别？

### 先给结论

这道题最容易讲乱，因为分页和分段都在做“逻辑地址到物理地址的转换”。

但你记住一句话，基本就不会乱：

- **分页是按固定大小切，重点是方便内存管理**
- **分段是按逻辑模块切，重点是贴合程序结构**

也就是说，分页更像操作系统站在“怎么更好分配内存”的角度想问题；分段更像程序员站在“代码本来就分成不同功能块”的角度想问题。

### 用一个类比先记住

把一本书交给两个人处理：

第一个人按“每 10 页一刀”去切，这就是分页。

```text
第 1 份：1~10 页
第 2 份：11~20 页
第 3 份：21~30 页
```

它不关心你是不是刚好切在章节边界上，只在乎每份大小一样，方便管理。

第二个人按“目录章节”去切，这就是分段。

```text
第 1 段：前言
第 2 段：第一章
第 3 段：第二章
第 4 段：附录
```

每一段长短都可能不一样，但每段的意义非常明确。

### 区别 1：划分方式不同

分页是把程序和内存都切成**固定大小**的小块。

- 程序里的块叫页
- 内存里的块叫页框
- 页大小是固定的

分段是把程序按**逻辑功能**划分成不同的段。

- 比如代码段、数据段、栈段
- 每个段大小可以不一样
- 每个段都有自己的起始地址和长度

### 区别 2：地址结构不同

分页的逻辑地址通常是一维的，本质上是一个连续地址，只不过系统会把它拆成：

```text
页号 + 页内偏移
```

分段的逻辑地址更像二维地址：

```text
段号 + 段内偏移
```

因为你访问的不是“连续空间里的第几个字节”，而是“第几个逻辑段里的第几个字节”。

### 区别 3：目的不同

分页的重点是：

- 让内存分配更规整
- 减少外部碎片
- 提高内存利用率

分段的重点是：

- 让内存划分更符合程序逻辑
- 方便共享
- 方便保护

比如代码段可以设置为只读，数据段可以读写，这种保护在分段思路下就很自然。

### 区别 4：碎片问题不同

分页因为大小固定，**外部碎片少**，但最后一页可能装不满，所以会有**内部碎片**。

分段因为每段大小不同，**容易出现外部碎片**。

这个点特别常考，别记反了。

### 区别 5：对程序员是否“有感知”不同

分页通常对程序员是透明的。

你写代码时一般不会关心“我这个变量在第几页”，这些都是操作系统和硬件在背后处理。

分段更贴近程序本身的结构。

因为程序天然就有代码、数据、栈这些不同逻辑区域，所以分段更容易让人从程序结构的角度去理解。

### 对比总结

| 对比点 | 分页 | 分段 |
|--------|------|------|
| 划分依据 | 固定大小 | 逻辑模块 |
| 大小 | 页大小固定 | 段大小可变 |
| 地址形式 | 页号 + 页内偏移 | 段号 + 段内偏移 |
| 主要目的 | 提高内存利用率 | 满足程序逻辑和保护需求 |
| 碎片问题 | 可能有内部碎片 | 容易有外部碎片 |
| 共享和保护 | 相对没那么直观 | 更方便实现 |

### ⚠️ 面试易错点

- **不要只背“分页固定、分段可变”**：这只是表面区别，更核心的是“分页偏管理，分段偏逻辑”。
- **分页不是按功能分的**：它不关心这一块是代码还是数据，只关心大小统一。
- **分段不是为了提升内存利用率而设计的**：它更主要是为了让程序的不同逻辑部分独立管理。
- **分页主要解决外部碎片问题，分段反而容易产生外部碎片**。
- **分页和分段都要做地址映射**：只是一个查页表，一个查段表。

### 面试回答模板

分页和分段都是内存管理方式，但它们的出发点不一样。

分页是把程序和物理内存划分成固定大小的页和页框，通过页表完成映射，主要目的是提高内存利用率，减少外部碎片。它对程序员通常是透明的，不过可能会产生内部碎片。

分段是按照程序的逻辑结构来划分，比如代码段、数据段、栈段等，通过段表记录每个段的基址和段长。它的优点是更符合程序逻辑，方便共享和保护，但由于段大小不固定，容易产生外部碎片。

所以可以简单理解为：分页偏内存管理效率，分段偏程序逻辑组织。

### 面试简答版

分页是按固定大小切，分段是按逻辑模块切。分页主要为了提高内存利用率、减少外部碎片；分段主要为了满足程序逻辑、共享和保护需求。

分页大小固定，通常是一维地址；分段大小可变，通常是段号加段内偏移。分页容易有内部碎片，分段容易有外部碎片。

## 4. 什么是交换空间？

### 先说场景

内存是很宝贵的，但程序一多，物理内存总有不够用的时候。

这时候操作系统不会立刻让程序全崩掉，而是会想办法把**暂时不常用的数据先挪出去**，把内存位置腾出来给当前更需要运行的内容。

这个“临时挪出去”的地方，通常就是磁盘上的一块区域，也就是**交换空间**。

### 大白话解释

可以把内存想象成你的书桌，磁盘想象成旁边的大柜子。

书桌拿东西很快，但地方小；柜子地方大，但拿东西慢。

如果你桌上摆满了资料，你暂时不用的那几本书，就可以先放进柜子里，等需要时再拿回来。

```text
内存 = 书桌
交换空间 = 柜子里的临时存放区
交换 = 把东西从桌上搬到柜子里，或者再搬回来
```

所以交换空间不是让程序“消失”了，而是把一部分暂时不用的内容先放到更慢但更大的地方。

### 什么是交换空间，什么是交换？

这两个词很容易混。

- **交换空间（swap space）**：磁盘上的一块区域，专门用来临时存放从内存换出去的页
- **交换（swapping）**：把内存里的内容换到磁盘，或者再从磁盘换回内存的过程

你可以这样记：

```text
交换空间 = 地方
交换 = 动作
```

### 为什么需要交换空间？

最核心的原因就一个：**物理内存不够。**

当内存吃紧时，操作系统会把一些暂时不活跃的页换到交换空间里，这样就能释放出内存给当前活跃的程序使用。

常见场景有：

- 某些程序开着但长时间没操作
- 程序启动时加载了很多内容，后面其中一部分很久都用不到
- 系统同时运行的进程太多，RAM 不够

### 它和分页是什么关系？

交换空间通常是和分页机制一起工作的。

因为系统管理内存时，本来就常常是按“页”来管理，所以换出去、换回来的单位，通常也是**页**。

也就是说：

```text
页在内存里放不下
-> 某些不常用页被换到 swap 空间
-> 需要时再换回内存
```

所以交换空间本质上不是独立的一套新机制，而是虚拟内存管理里的一个重要配套手段。

### 它和虚拟内存是什么关系？

这个也是高频易混点。

你可以先这样理解：

- **物理内存**：真正的 RAM
- **交换空间**：磁盘上拿来临时顶内存压力的区域
- **虚拟内存**：操作系统给每个进程“看到的一套更大、更连续”的内存抽象

交换空间是虚拟内存能够工作的一个重要补充，但**交换空间不等于虚拟内存本身**。

虚拟内存是整体机制，交换空间只是其中一个帮手。

### 交换空间的优点和代价

它的优点很直接：

- 能缓解内存不足
- 让系统不至于因为 RAM 紧张立刻崩掉
- 能让更多进程同时存在

但代价也很明显：

- 磁盘速度远慢于内存
- 如果频繁换入换出，系统会明显变卡

这就像你一边干活一边不停从书桌跑去柜子里拿资料，肯定比直接在桌上翻慢得多。

如果系统大部分时间都在忙着换页，而不是忙着真正执行程序，这种现象就叫 **抖动（Thrashing）**。

### ⚠️ 面试易错点

- **交换空间在磁盘上，不在内存里**。
- **交换空间不等于虚拟内存**：它只是虚拟内存机制的一部分。
- **交换通常按页进行，不一定是整个进程一起搬**。
- **有 swap 不代表性能更好**：swap 的本质是“救场”，不是让磁盘变成和内存一样快。
- **频繁使用 swap 往往说明内存压力很大**，系统可能已经开始变慢。

### 面试回答模板

交换空间是磁盘上的一块区域，用来临时存放从内存中换出的页。当物理内存不足时，操作系统会把暂时不常用的页换到交换空间中，释放 RAM 给当前更活跃的程序使用，这个过程叫交换。

交换空间通常和分页机制一起工作，因为换入换出的基本单位往往是页。它可以缓解内存不足，让系统支持更多进程继续运行，但由于磁盘速度远慢于内存，如果频繁发生交换，系统性能会明显下降，严重时还会出现抖动。

所以可以把它理解成虚拟内存机制中的一个辅助区域，用来在内存紧张时临时“托底”。

### 面试简答版

交换空间就是磁盘上的一块区域，用来存放从内存换出去的页；交换就是把页在内存和磁盘之间来回搬的过程。

它的作用是缓解物理内存不足，但因为磁盘比内存慢很多，所以频繁使用交换空间会导致系统变卡。
