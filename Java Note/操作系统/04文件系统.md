# 文件系统

## 1. 硬链接和软链接有什么区别？

### 先说场景

有时候我们想给一个文件起多个“入口”。

比如：

- 一个文件想在多个目录里都能访问
- 想做一个快捷入口
- 想保留原文件位置不变，但从别的地方也能打开

这时候就会遇到硬链接和软链接。

### 大白话解释

可以先这样理解：

- **硬链接**：相当于同一个人有两个正式名字，两个名字都直接指向同一个实体
- **软链接**：相当于你手里有一张写着别人住址的小纸条，靠这张纸条去找原目标

所以它们最大的区别就是：

- 硬链接更像“再起一个并列名字”
- 软链接更像“做一个跳转指针”

### 什么是硬链接？

硬链接本质上是给同一个文件内容再增加一个目录项。

多个硬链接共享同一个 inode，也就是说，它们本质上指向的是同一个文件实体。

这意味着：

- 改其中一个，另一个看到的内容也会变
- 删掉其中一个名字，文件不一定消失
- 只有所有硬链接都删掉，并且没有进程占用时，文件内容才真正被回收

### 什么是软链接？

软链接可以理解成一个“特殊文件”，里面存的是目标文件的路径。

访问软链接时，系统会根据这条路径再去找真正的目标文件。

所以软链接更像 Windows 里的快捷方式。

### 两者区别

| 对比点 | 硬链接 | 软链接 |
|--------|--------|--------|
| 指向方式 | 直接指向同一个 inode | 指向目标路径 |
| 删除源文件后 | 只要还有别的硬链接，文件还在 | 源文件删了，链接就失效 |
| 能否跨文件系统 | 不能 | 可以 |
| 能否链接目录 | 一般不能 | 可以（很多系统支持） |
| 本质 | 同一个文件的多个名字 | 一个单独的链接文件 |

### 为什么硬链接不能跨文件系统？

因为 inode 是文件系统内部的概念。

硬链接要直接指向 inode，而不同文件系统里的 inode 不是同一套管理范围，所以没法跨过去直接连。

### 为什么软链接更灵活？

因为它只记录路径。

路径本身不关心底层 inode 在哪个文件系统，所以它更容易跨目录、跨文件系统使用。

### ⚠️ 面试易错点

- **硬链接不是“复制一份文件”**，它只是多了一个文件名入口。
- **软链接不是目标文件本身**，它只是保存目标路径。
- **删掉源文件后，硬链接通常还能访问，软链接通常会失效**。
- **硬链接一般不能跨文件系统，也不能随便链接目录**。

### 面试回答模板

硬链接和软链接的核心区别在于指向方式不同。硬链接是给同一个文件增加新的目录项，多个硬链接共享同一个 inode，本质上是同一个文件的多个名字；软链接则是一个单独的链接文件，里面保存的是目标文件的路径。

因此，删除原文件后，如果还有其他硬链接，文件内容依然存在；而软链接通常会失效。硬链接不能跨文件系统，软链接可以跨文件系统，也更灵活。

### 面试简答版

硬链接是同一个文件的多个名字，直接指向同一个 inode；软链接是保存目标路径的链接文件。

硬链接删一个名字不影响文件本体，软链接删掉源文件后通常会失效。
