# 辅助工具类

## 1. Array 和 ArrayList 有何区别？

### 先说场景

如果你要存一个班 **50 个学生**的成绩，数量确定不会变 → **Array**
如果存的是"用户购物车商品"，不知道用户会加几件 → **ArrayList**

核心区别就一句话：**Array 长度固定，ArrayList 动态扩容**。

### 对比一览

| | Array | ArrayList |
|---|---|---|
| **长度** | 固定，创建后不能改 | 动态扩容（默认 1.5 倍） |
| **基本类型** | ✅ 直接存 `int[]` | ❌ 必须用包装类 `Integer`（自动装箱让你感觉像能存） |
| **方法** | 只有 `length` 属性 | `add`、`remove`、`contains` 等 |
| **性能** | 快，直接内存操作 | 稍慢（扩容拷贝 + 装箱开销） |
| **泛型** | ❌ `List<int>` 不行 | ✅ `List<Integer>` |

### 简单代码

```java
// Array —— 长度固定
int[] arr = new int[3];
arr[0] = 1;
arr[1] = 2;
arr[2] = 3;
// arr[3] = 4;  // ❌ ArrayIndexOutOfBoundsException

// ArrayList —— 随便加
ArrayList<Integer> list = new ArrayList<>();
list.add(1);
list.add(2);
list.add(3);
list.add(4);  // ✅ 自动扩容
```

**输出：**
```
arr.length = 3
list.size() = 4
```

### 深入理解

#### 1. 动态扩容（面试重点）

`ArrayList` 底层还是数组：

```java
Object[] elementData = new Object[10];  // 默认初始容量 10
```

当第 11 个元素进来时：
1. 新创建一个 **1.5 倍**大小的新数组
2. 把老数组**全部拷贝**过去
3. 老数组等垃圾回收

⚠️ **坑**：如果你知道大概要存 1000 个，给个初始容量：

```java
ArrayList<Integer> list = new ArrayList<>(1000);  // 避免多次扩容
```

每次扩容都是 **O(n)** 的拷贝操作，频繁触发很伤性能。

#### 2. 基本类型装箱开销

```java
int[] arr = new int[3];                    // ✅ 堆上连续内存，存的是值
ArrayList<Integer> list = new ArrayList<>(); // ✅ 但存的是引用（包装类对象）
```

因为有**自动装箱（autoboxing）**，写起来没区别：

```java
list.add(42);  // 编译器变成 list.add(Integer.valueOf(42));
```

⚠️ **坑**：大量 `int` 数据别用 `ArrayList`，每个 `int` 变成 `Integer` 对象，内存开销大约 **4 倍**。

#### 3. 性能差距

```java
int[] arr = new int[100000];
ArrayList<Integer> list = new ArrayList<>(100000);

long start = System.nanoTime();
for (int i = 0; i < 100000; i++) arr[i] = i;
System.out.println("Array: " + (System.nanoTime() - start) + "ns");

start = System.nanoTime();
for (int i = 0; i < 100000; i++) list.add(i);
System.out.println("ArrayList: " + (System.nanoTime() - start) + "ns");
```

Array 快得多，但日常写代码**基本感觉不到**，除非几十万级别的循环。

### 面试回答模板

> Array 固定长度，ArrayList 动态扩容（1.5 倍 + 拷贝）。Array 能存基本类型，ArrayList 不能但自动装箱让你感觉能存。日常用 ArrayList 更多，因为灵活；性能敏感、数量确定的场景用 Array。

### 面试简答版

| Array | ArrayList |
|---|---|
| 固定长度 | 自动扩容（1.5 倍） |
| 支持基本类型 | 只支持对象（包装类） |
| 只有 `length` 属性 | `add`/`remove`/`contains` 等方法 |
| 性能好 | 有扩容和装箱开销 |
| `int[] arr = new int[3]` | `ArrayList<Integer> list = new ArrayList<>()` |

## 2. 如何实现 Array 和 List 之间的转换？

两个方向，记住两个方法就行：

### Array → List：`Arrays.asList()`

```java
String[] arr = {"a", "b", "c"};
List<String> list = Arrays.asList(arr);
```

⚠️ **天坑**：这个 `list` **不是普通的 ArrayList**，它返回的是 `Arrays` 内部的一个**固定大小视图**，底层还是原来的数组。不能 `add`、不能 `remove`：

```java
list.add("d");  // ❌ UnsupportedOperationException
```

想要真正的可变列表，包一层：

```java
List<String> realList = new ArrayList<>(Arrays.asList(arr));
realList.add("d");  // ✅ 没问题
```

### List → Array：`list.toArray()`

```java
List<String> list = new ArrayList<>();
list.add("a");
list.add("b");

// 方式一：返回 Object[]
Object[] objArr = list.toArray();

// 方式二（推荐）：指定类型，传入一个空数组
String[] arr = list.toArray(new String[0]);
```

**为什么推荐 `new String[0]`？** Java 官方在 JDK 6 之后做了优化——如果传入的数组不够大，会反射重新创建一个，所以传空数组性能反而更好（不用浪费空间）。

### 面试回答模板

> Array 转 List 用 `Arrays.asList()`，但返回的是固定大小视图，不能增删，要可变就 `new ArrayList<>(Arrays.asList(...))`。List 转 Array 用 `list.toArray(new T[0])`，传空数组就行，JDK 有优化。

### 面试简答版

| 方向 | 方法 | 注意 |
|------|------|------|
| **Array → List** | `Arrays.asList(arr)` | 返回固定大小视图，不能 add/remove |
| **Array → 可变 List** | `new ArrayList<>(Arrays.asList(arr))` | 包一层才是真 ArrayList |
| **List → Array** | `list.toArray(new T[0])` | JDK 优化过，传空数组即可 |

## 3. Comparable 和 Comparator 有何区别？

### 先说场景

你有一堆学生要排序——按学号？按年龄？按姓名？

这就涉及到两个角色：
- **学生自己**能不能告诉别人"我默认按什么排"？
- 还是需要一个**外部裁判**，在不同场合按不同规则排？

**Comparable** 就是"自己定规则"，**Comparator** 就是"找裁判定规则"。

### 一句话核心

| | Comparable | Comparator |
|---|---|---|
| **角色** | 内部比较器（类自己实现） | 外部比较器（独立实现） |
| **方法** | `compareTo(T o)` | `compare(T a, T b)` |
| **包** | `java.lang` | `java.util` |
| **定义** | 自然排序（默认规则） | 临时/定制排序 |
| **修改代价** | 要改原类源码 | 不用改原类 |
| **灵活度** | 一个类只有一种 | 可以写无数种 |
| **函数式** | ❌ 不是函数式接口 | ✅ 可用 lambda |

### 比喻

- **Comparable**：每个人都有身份证号，这就是"自然顺序"。问谁先谁后？按身份证号比就行，不用别人教。
- **Comparator**：今天按身高排、明天按年龄排、后天按名字拼音排——这些是外部规则，跟人本身没关系。

### 简单代码

```java
// ---- Comparable：类自己实现 ----
public class Student implements Comparable<Student> {
    String name;
    int age;

    @Override
    public int compareTo(Student o) {
        return this.age - o.age;  // 默认按年龄排
    }
}

List<Student> list = new ArrayList<>();
Collections.sort(list);  // ✅ 自动用 Comparable 的 compareTo

// ---- Comparator：外部裁判 ----
Comparator<Student> byName = (a, b) -> a.name.compareTo(b.name);
Collections.sort(list, byName);  // ✅ 传入外部比较器

// 还可以随意换规则
Collections.sort(list, (a, b) -> b.age - a.age);  // 按年龄降序
```

### 实际应用场景

```java
// String、Integer 等包装类自带 Comparable
List<Integer> nums = Arrays.asList(3, 1, 2);
Collections.sort(nums);  // ✅ 按自然顺序升序

// 降序就用 Comparator
Collections.sort(nums, (a, b) -> b - a);  // ✅ 外部规则

// 多级排序：先按年龄，再按姓名
Comparator<Student> multi = Comparator
    .comparingInt((Student s) -> s.age)
    .thenComparing(s -> s.name);
```

### 面试回答模板

> Comparable 是内部比较器，定义在类自身里，实现 `compareTo()` 定义自然排序。Comparator 是外部比较器，单独实现 `compare()` 定义临时排序。Comparable 在 `java.lang`，Comparator 在 `java.util`。一个类只能有一种 Comparable，但可以有多种 Comparator。Comparator 是函数式接口，可以用 lambda 简洁书写。

### 面试简答版

| | Comparable | Comparator |
|---|---|---|
| 包 | `java.lang` | `java.util` |
| 方法 | `compareTo(T o)` | `compare(T a, T b)` |
| 谁实现 | 类自己 | 外部独立写 |
| 用途 | 自然排序 | 定制排序 |
| 数量 | 一个类一种 | 可以写无数种 |
| lambda | ❌ 不行 | ✅ 可以 |

## 4. Collection 和 Collections 有什么区别？

### 先说场景

面试官故意问的——两个词发音一样，但一个是接口，一个是类。看你是不是真清楚。

### 一句话核心

| | `Collection` | `Collections` |
|---|---|---|
| **本质** | **接口**，集合体系的根 | **工具类**，全是静态方法 |
| **包** | `java.util.Collection` | `java.util.Collections` |
| **作用** | 定义集合的顶层行为 | 提供操作集合的工具方法 |
| **实例化** | 不能直接 new，靠子类 | 构造器私有，不能 new |
| **你认识的** | `List`、`Set`、`Queue` | `sort`、`reverse`、`shuffle`、`unmodifiableList` |

### 比喻

- **Collection** = "餐厅"这个概念——定义了餐厅应该有什么（菜单、座位、服务员）。`List`、`Set` 是不同类型的餐厅。
- **Collections** = "餐厅工具箱"——摆桌子（`sort`）、打乱椅子（`shuffle`）、加玻璃罩子不让碰（`unmodifiableList`）。全是静态方法，拿来就用。

### 简单代码

```java
// ---- Collection：接口，定义行为 ----
Collection<String> c = new ArrayList<>();
c.add("a");
c.add("b");
System.out.println(c.size());  // 2

// ---- Collections：工具类，静态方法 ----
List<String> list = new ArrayList<>(Arrays.asList("c", "a", "b"));
Collections.sort(list);          // [a, b, c]
Collections.reverse(list);       // [c, b, a]
Collections.shuffle(list);       // 随机打乱

// 返回只读视图（跟 Arrays.asList 一样不能增删）
List<String> readonly = Collections.unmodifiableList(list);
// readonly.add("x");  // ❌ UnsupportedOperationException
```

### 常用工具方法

| 方法 | 作用 |
|------|------|
| `sort()` | 排序（要求元素实现 Comparable 或传入 Comparator） |
| `reverse()` | 反转顺序 |
| `shuffle()` | 随机打乱 |
| `binarySearch()` | 二分查找（前提：已排序） |
| `unmodifiableList/Set/Map()` | 返回只读视图，不能修改 |
| `synchronizedList/Set/Map()` | 返回线程安全版本 |
| `emptyList/Set/Map()` | 返回空集合，省内存 |
| `singletonList/Set/Map()` | 返回只有一个元素的集合 |

### 面试回答模板

> Collection 是接口，集合体系的顶层接口，List、Set、Queue 都继承它。Collections 是工具类，所有方法都是 static 的，提供 sort、reverse、unmodifiableList 等工具方法。名字很像、读音一样，但一个是接口一个是类，面试常考这个区分。

### 面试简答版

| | Collection | Collections |
|---|---|---|
| 本质 | 接口 | 工具类 |
| 作用 | 定义集合行为 | 提供操作集合的静态方法 |
| 实例化 | 靠实现类（List、Set 等） | 不能 new |
| 例子 | `Collection<String> c = new ArrayList<>()` | `Collections.sort(list)` |

## 5. TreeMap/TreeSet 和 Collections.sort() 如何比较元素？

### 先说场景

你往 `TreeSet` 里加元素、往 `TreeMap` 里放键、或者调 `Collections.sort()`——Java 怎么知道谁前谁后？

答案：**它们自己不会比，全依赖 Comparable 和 Comparator。** 也就是 [[#3 Comparable 和 Comparator 有何区别]] 那两个机制的实际应用。

### 统一规则

| | TreeMap / TreeSet | Collections.sort() |
|---|---|---|
| **比较时机** | 插入时（每次 `add`/`put`） | 主动调用时（一次性） |
| **Comparable** | 元素未实现 → 运行时抛 `ClassCastException` | 元素未实现 → 编译报错 |
| **Comparator** | 构造时传入 | 作为参数传入 |
| **优先级** | 构造传了 Comparator 就用它，否则用 Comparable | 看你调哪个重载 |
| **相等判定** | `compareTo`/`compare` 返回 0 → 视为同一个元素 | 不影响去重 |

### TreeMap / TreeSet：插入时就比

```java
// 方式一：元素自身实现 Comparable
public class Student implements Comparable<Student> {
    String name;
    int age;
    public int compareTo(Student o) {
        return this.age - o.age;  // 自然顺序：按年龄
    }
}
TreeSet<Student> set = new TreeSet<>();  // 走 Comparable

// 方式二：构造时传入 Comparator
TreeSet<Student> set = new TreeSet<>(
    (a, b) -> a.name.compareTo(b.name)   // 定制顺序：按姓名
);
```

⚠️ **重点坑**：TreeSet/TreeMap **用 compareTo 判断重复**，不是 `equals()`。compare 返回 0 就认为是同一个元素：

```java
TreeSet<String> set = new TreeSet<>((a, b) -> 0);  // 所有元素都"相等"
set.add("a");
set.add("b");
set.add("c");
System.out.println(set.size());  // 1（a、b、c 被认为一样）
```

### Collections.sort()：主动调才比

```java
// 重载一：用元素的 Comparable
Collections.sort(list);  // 要求 T implements Comparable

// 重载二：传入 Comparator
Collections.sort(list, (a, b) -> a.age - b.age);

// JDK 8+ 还可以直接调 List.sort()
list.sort((a, b) -> a.age - b.age);
```

底层是 **TimSort**（归并排序 + 插入排序的混合），JDK 7+ 开始用，稳定排序，O(n log n)。

### 面试回答模板

> 它们都依赖 Comparable 和 Comparator。TreeMap/TreeSet 在插入时比较，构造时可传 Comparator，没传就用元素的 Comparable，都没给就抛异常。而且它们用 compareTo 而不是 equals 判断重复。Collections.sort() 也有两个重载，底层是 TimSort。

### 面试简答版

| 容器 | 比较时机 | Comparable | Comparator | 底层算法 |
|------|---------|-----------|-----------|---------|
| TreeMap/TreeSet | 插入时 | ✅ 默认使用 | ✅ 构造传入 | 红黑树 O(log n) |
| Collections.sort() | 调用时 | ✅ 一种重载 | ✅ 另一种重载 | TimSort O(n log n) |
