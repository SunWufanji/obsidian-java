# 146. LRU 缓存

## 题目

设计并实现一个满足 `LRU (Least Recently Used)` 约束的数据结构。

实现 `LRUCache` 类：

- `LRUCache(int capacity)`：以正整数 `capacity` 作为容量初始化缓存
- `int get(int key)`：如果关键字 `key` 存在于缓存中，则返回关键字的值，否则返回 `-1`
- `void put(int key, int value)`：如果关键字已存在，则更新其值；如果不存在，则插入该组 `key-value`。当缓存容量达到上限时，应删除最久未使用的关键字

要求：

- `get` 和 `put` 都必须在平均 `O(1)` 时间复杂度内完成

---

## 先理解题意

`LRU` 的意思是：

- 最近使用过的，保留
- 最久没使用过的，淘汰

例如容量是 `2`：

1. 放入 `(1,1)`，缓存中有 `1`
2. 放入 `(2,2)`，缓存中有 `2, 1`
3. 访问 `1`，此时 `1` 变成最近使用，顺序变成 `1, 2`
4. 放入 `(3,3)`，容量满了，要删掉最久没用的 `2`

所以这题要同时解决两件事：

1. 快速通过 `key` 找到值
2. 快速维护“最近使用顺序”

---

## 最优解：哈希表 + 双向链表

### 为什么这是最优解

题目要求 `get` 和 `put` 都要 `O(1)`。

如果只用哈希表：

- 可以快速找值
- 但没法快速知道谁最久没用

如果只用链表：

- 可以维护顺序
- 但找某个 key 需要遍历

所以需要两个结构配合：

- 哈希表：`O(1)` 找节点
- 双向链表：`O(1)` 删节点、插头部、删尾部

---

## 核心思路

维护两个结构：

### 1. 哈希表

```text
map[key] = 对应的链表节点
```

作用：

- 通过 `key` 直接找到节点

### 2. 双向链表

链表每个节点保存：

- `key`
- `value`
- `prev/pre`
- `next`

约定：

- 头部附近表示最近使用
- 尾部附近表示最久未使用

通常再加两个虚拟节点：

- `head`
- `tail`

这样插删更统一，不容易写出一堆边界判断。

---

## 最优解代码

```java
import java.util.HashMap;
import java.util.Map;

class LRUCache {
    class Node {
        int key;
        int value;
        Node prev;
        Node next;

        Node() {}

        Node(int key, int value) {
            this.key = key;
            this.value = value;
        }
    }

    private final int capacity;
    private final Map<Integer, Node> map;
    private final Node head;
    private final Node tail;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.map = new HashMap<>();
        this.head = new Node();
        this.tail = new Node();

        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        Node node = map.get(key);
        if (node == null) {
            return -1;
        }

        moveToHead(node);
        return node.value;
    }

    public void put(int key, int value) {
        Node node = map.get(key);

        if (node != null) {
            node.value = value;
            moveToHead(node);
            return;
        }

        Node newNode = new Node(key, value);
        map.put(key, newNode);
        addToHead(newNode);

        if (map.size() > capacity) {
            Node removed = removeTail();
            map.remove(removed.key);
        }
    }

    private void addToHead(Node node) {
        node.prev = head;
        node.next = head.next;
        head.next.prev = node;
        head.next = node;
    }

    private void removeNode(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void moveToHead(Node node) {
        removeNode(node);
        addToHead(node);
    }

    private Node removeTail() {
        Node node = tail.prev;
        removeNode(node);
        return node;
    }
}
```

## 同思路写法：你的手写版

这份代码和上面的标准模板版本质一样，都是：

- 哈希表 + 双向链表
- `get/put` 都是 `O(1)`

区别是这份更贴近你自己的书写习惯，复习时可以两份对照着看。

```java
import java.util.*;
class LRUCache {
    class Node {
        int key;
        int value;
        Node pre;
        Node next;
        public Node(){}
        public Node(int key, int value) {
            this.key = key;
            this.value = value;
        }
    }

    private Map<Integer, Node> cache;
    private int capacity;
    private Node head;
    private Node tail;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        cache = new HashMap<>();
        head = new Node();
        tail = new Node();
        // 由于put操作会使用到head.next, 此处需事先声明
        head.next = tail;
        tail.pre = head;
    }
    public int get(int key) {
        if (cache.containsKey(key)) {
            Node node = cache.get(key);
            removeNode(node);
            addNodeToHead(node);
            return node.value;
        }
        return -1;
    }
    public void put(int key, int value) {
        if (cache.containsKey(key)) {
            Node node = cache.get(key);
            node.value = value;
            removeNode(node);
            addNodeToHead(node);
        } else {
            if (cache.size() == capacity) {
                Node tailPre = tail.pre;
                removeNode(tailPre);
                cache.remove(tailPre.key);
            }
            Node newNode = new Node(key, value);
            cache.put(key, newNode);
            addNodeToHead(newNode);
        }
    }

   private void removeNode(Node node) {
        node.pre.next = node.next;
        node.next.pre = node.pre;
   }
   private void addNodeToHead(Node node) {
        node.next = head.next;
        node.next.pre = node;
        head.next = node;
        node.pre = head;
   }
}

public class Main{
    public static void main(String[] args) {
        LRUCache cache = new LRUCache(2);
        cache.put(1, 1);
        cache.put(2, 2);
        System.out.println(cache.get(1)); // 输出 1
        cache.put(3, 3); // 键2(2,2)被淘汰
        System.out.println(cache.get(2)); // 输出 -1
        cache.put(4, 4); // 键1被淘汰
        System.out.println(cache.get(1)); // 输出 -1
    }
}
```

```java

    if (cache.containsKey(key)) {
```

#### 返回值


- 返回该 key 对应的值

#### 不存在的情况

```java

```

- 表示没找到

### 7. `put` 方法

```java
public void put(int key, int value) {
    if (cache.containsKey(key)) {
        Node node = cache.get(key);
        node.value = value;
        removeNode(node);
        addNodeToHead(node);
    } else {
        if (cache.size() == capacity) {
            Node tailPre = tail.pre;
            removeNode(tailPre);
            cache.remove(tailPre.key);
        }
        Node newNode = new Node(key, value);
        cache.put(key, newNode);
        addNodeToHead(newNode);
    }
}
```

`put` 要分两大类情况。

#### 情况 1：key 已存在

```java
if (cache.containsKey(key)) {
```

说明：

- 不是新插入
- 而是更新旧值

```java
Node node = cache.get(key);
```

- 先拿到旧节点

```java
node.value = value;
```

- 更新值

```java
removeNode(node);
addNodeToHead(node);
```

- 更新后也算最近使用
- 所以要移动到头部

#### 情况 2：key 不存在

```java
else {
```

说明：

- 要插入一个全新的缓存项

##### 先判断容量是否已满

```java
if (cache.size() == capacity) {
```

- 如果当前元素个数已经等于容量
- 插入新元素前，必须先淘汰一个旧元素

##### 找到最久未使用节点

```java
Node tailPre = tail.pre;
```

- `tail.pre` 就是尾部前一个节点
- 它正是“最久未使用”的真实节点

##### 从链表删掉

```java
removeNode(tailPre);
```

- 先从双向链表中删除它

##### 再从哈希表删掉

```java
cache.remove(tailPre.key);
```

- 不能只删链表，不删哈希表
- 否则 `cache` 里还保留着旧数据，会不一致

##### 创建新节点

```java
Node newNode = new Node(key, value);
```

- 用新的 `key-value` 创建节点

##### 放入哈希表

```java
cache.put(key, newNode);
```

- 建立 `key -> 节点` 映射

##### 插到链表头部

```java
addNodeToHead(newNode);
```

- 新插入的元素一定是最近使用

---

### 8. `removeNode`

```java
private void removeNode(Node node) {
    node.pre.next = node.next;
    node.next.pre = node.pre;
}
```

这是双向链表删除一个已知节点的标准写法。

假设链表中有：

```text
A <-> node <-> B
```

执行后变成：

```text
A <-> B
```

第一句：

```java
node.pre.next = node.next;
```

- 让前一个节点跳过当前节点，直接指向后一个节点

第二句：

```java
node.next.pre = node.pre;
```

- 让后一个节点的前驱也跳过当前节点，直接指向前一个节点

这样当前节点就从链表中被摘掉了。

---

### 9. `addNodeToHead`

```java
private void addNodeToHead(Node node) {
    node.next = head.next;
    node.next.pre = node;
    head.next = node;
    node.pre = head;
}
```

这是头插法。

目标是把 `node` 插入到：

```text
head 和 原来的第一个真实节点之间
```

假设原来是：

```text
head <-> first
```

插入后要变成：

```text
head <-> node <-> first
```

逐句理解：

```java
node.next = head.next;
```

- 先让新节点的后继指向原来的第一个节点

```java
node.next.pre = node;
```

- 再让原来的第一个节点的前驱改成新节点

```java
head.next = node;
```

- 再让 head 的后继改成新节点

```java
node.pre = head;
```

- 最后让新节点前驱指向 head

这四句顺序可以稍有不同，但核心是把四条指针都接正确。

## 你的代码整体在做什么

你这份代码其实可以压缩成一句话：

- 哈希表负责快速定位节点
- 双向链表负责维护最近使用顺序
- `get` 命中就移到头部
- `put` 更新或插入后也移到头部
- 超容量就删除尾部前一个节点

这就是 LRU 的完整思想。

## 代码执行流程

假设：

```text
capacity = 2
```

### 1. `put(1, 1)`

- 新建节点 `1`
- 放到头部

链表：

```text
head <-> 1 <-> tail
```

### 2. `put(2, 2)`

- 新建节点 `2`
- 放到头部

链表：

```text
head <-> 2 <-> 1 <-> tail
```

说明：

- `2` 最近使用
- `1` 更久一点

### 3. `get(1)`

- 从哈希表找到节点 `1`
- 把 `1` 移动到头部

链表：

```text
head <-> 1 <-> 2 <-> tail
```

### 4. `put(3, 3)`

- 新建节点 `3`
- 插到头部

临时链表：

```text
head <-> 3 <-> 1 <-> 2 <-> tail
```

容量超了，删除尾部前一个节点 `2`

最终：

```text
head <-> 3 <-> 1 <-> tail
```

## 时间复杂度

### `get`

- 哈希表查找：`O(1)`
- 双向链表删除并插入头部：`O(1)`

所以：

```text
O(1)
```

### `put`

- 哈希表查找：`O(1)`
- 插入新节点：`O(1)`
- 删除尾节点：`O(1)`
- 哈希表删除：`O(1)`

所以：

```text
O(1)
```

## 空间复杂度

哈希表和链表里最多存 `capacity` 个真实节点，所以空间复杂度是：

```text
O(capacity)
```

也常写作：

```text
O(n)
```

## 易错点

### 1. 节点里一定要存 `key`

淘汰节点时要从哈希表删除：

```java
cache.remove(tailPre.key);
```

如果节点里没有 `key`，这里就删不掉。

### 2. 更新已有 key 时不要新建节点

如果 key 已存在：

- 直接更新 `value`
- 移到头部

不要再创建新节点，否则链表和哈希表容易乱。

### 3. 删除尾节点后要同步删哈希表

只删链表，不删哈希表，会导致数据不一致。

### 4. 头尾虚拟节点很重要

没有虚拟节点，边界处理会麻烦很多。

---

## 面试怎么讲

可以这样说：

> 这题要求 `get` 和 `put` 都在 `O(1)` 内完成，所以单独使用哈希表或者链表都不够。  
> 我用哈希表 `key -> node` 做 `O(1)` 查找，用双向链表维护访问顺序。  
> 链表头部表示最近使用，尾部表示最久未使用。  
> `get` 命中后把节点移到头部；`put` 时如果 key 已存在就更新并移动到头部，不存在就新建节点插入头部。  
> 如果容量超了，就删除尾部节点，并同步从哈希表删除。  
> 因为哈希表查找和双向链表插删都是 `O(1)`，所以整体满足题目要求。

## 解法二：Java 内置 `LinkedHashMap`

### 思路

Java 的 `LinkedHashMap` 可以维护访问顺序。

如果这样写：

```java
new LinkedHashMap<>(capacity, 0.75f, true)
```

最后一个参数 `true` 表示按访问顺序维护。

这样：

- `get`
- 或者对已存在 key 执行 `put`

对应元素都会变成“最近访问”。

### 代码

```java
import java.util.LinkedHashMap;
import java.util.Map;

class LRUCache {
    private final int capacity;
    private final LinkedHashMap<Integer, Integer> map;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.map = new LinkedHashMap<Integer, Integer>(capacity, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry<Integer, Integer> eldest) {
                return size() > LRUCache.this.capacity;
            }
        };
    }

    public int get(int key) {
        return map.getOrDefault(key, -1);
    }

    public void put(int key, int value) {
        map.put(key, value);
    }
}
```

### 特点

- 写法最短
- 工程里很好用
- 但如果面试要求手写底层结构，不如双向链表版本更能体现能力

## 解法三：固定数组映射 + 双向链表

### 思路

如果题目里的 `key` 范围比较小，而且是非负整数，那么可以不用 `HashMap`，直接开一个数组：

```java
Node[] map = new Node[10001];
```

这样：

- `map[key]` 就能直接拿到节点
- 本质上相当于“用数组代替哈希表”

双向链表部分不变，仍然负责维护最近使用顺序。

### 代码

```java
class Node {
    int key;
    int val;
    Node pre;
    Node next;

    Node(int key, int val) {
        this.key = key;
        this.val = val;
    }

    Node(int key, int val, Node pre, Node next) {
        this.key = key;
        this.val = val;
        this.pre = pre;
        this.next = next;
    }

    @Override
    public String toString() {
        return "(val=" + val + ")";
    }
}

class LRUCache {
    private Node[] map = new Node[10001];
    private int cap = 0;
    private int listSize = 0;
    private Node head = new Node(-1, -1);

    public LRUCache(int capacity) {
        cap = capacity;
        head.pre = head;
        head.next = head;
    }

    private void moveToHead(Node node) {
        node.next = head.next;
        head.next.pre = node;
        node.pre = head;
        head.next = node;
    }

    private void removeNode(Node node) {
        node.pre.next = node.next;
        node.next.pre = node.pre;
    }

    public int get(int key) {
        if (map[key] == null) return -1;
        removeNode(map[key]);
        moveToHead(map[key]);
        return map[key].val;
    }

    public void put(int key, int value) {
        if (map[key] == null) {
            listSize++;
            map[key] = new Node(key, value);
            moveToHead(map[key]);
            if (listSize > cap) {
                map[head.pre.key] = null;
                removeNode(head.pre);
                listSize--;
            }
        } else {
            removeNode(map[key]);
            moveToHead(map[key]);
            map[key].val = value;
        }
    }
}
```

### 特点

- 如果 `key` 范围很小，常数通常会更快
- 代码也能满足 `O(1)`
- 但它依赖“key 范围已知且不大”这个前提

### 局限

- 如果 `key` 很大，或者可能是负数，就不能这样直接开数组
- 通用性不如 `HashMap`
- 面试时如果题目没有明确给出适合开数组的 key 范围，优先答 `HashMap + 双向链表`

---

## 解法四：链表顺序模拟

### 思路

只用链表维护顺序：

- 头部表示最近使用
- 尾部表示最久未使用

`get` 时顺序查找，找到后移到头部。  
`put` 时顺序查找，存在就更新并移头，不存在就头插，超容量删尾。

### 问题

因为查找 key 需要遍历：

- `get`：`O(n)`
- `put`：最坏 `O(n)`

所以不满足题目要求，但适合帮助理解 LRU 的顺序维护本质。

---

## 三种解法对比

### 解法一：哈希表 + 双向链表

- 最优解
- 满足题目 `O(1)` 要求
- 面试最推荐

### 解法二：`LinkedHashMap`

- 写法最短
- 工程中很实用
- 面试中如果要求手写底层，不是最佳展示

### 解法三：固定数组映射 + 双向链表

- 常数可能更快
- 但依赖 key 范围较小
- 通用性不如 `HashMap`

### 解法四：链表顺序模拟

- 容易想到
- 适合理解
- 不满足题目要求

---

## 本题总结

这题是经典设计题，也是哈希表和双向链表配合的代表题。

重点记住：

1. 哈希表负责 `O(1)` 找节点
2. 双向链表负责 `O(1)` 调整顺序
3. 头部表示最近使用，尾部表示最久未使用
4. `get` 命中后移到头部
5. `put` 超容量时删尾部前一个真实节点

最优解的四个基本操作要会默写：

- `addToHead`
- `removeNode`
- `moveToHead`
- `removeTail`
