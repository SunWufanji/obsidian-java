# 48. 旋转图像

## 题目

给定一个 `n x n` 的二维矩阵 `matrix`，表示一个图像。请你将图像顺时针旋转 `90` 度。

要求：

- 必须 **原地** 旋转，不能使用另一个矩阵。

---

## 题意理解

这道题的关键有两个：

- 只能顺时针旋转 `90` 度
- 必须原地修改，不能新开一个同样大小的矩阵

例如：

```text
1 2 3
4 5 6
7 8 9
```

顺时针旋转后变成：

```text
7 4 1
8 5 2
9 6 3
```

---

## 核心思路

先记住一个位置变化规律：

原来在 `(i, j)` 的元素，旋转后会到：

```text
(j, n - 1 - i)
```

但如果直接按照这个规律移动元素，会覆盖原来的值，不太好处理。

所以更常用的方法是分成两步：

1. **先沿主对角线翻转**
2. **再将每一行左右翻转**

这样最终效果，正好等于顺时针旋转 `90` 度。

---

## 为什么这样做

### 第一步：主对角线翻转

主对角线翻转，就是交换：

```text
matrix[i][j] 和 matrix[j][i]
```

例如：

```text
1 2 3
4 5 6
7 8 9
```

对角线翻转后：

```text
1 4 7
2 5 8
3 6 9
```

这一步本质上是 **行列互换**。

### 第二步：每一行左右翻转

把上面的矩阵每一行再反转：

```text
1 4 7    ->    7 4 1
2 5 8    ->    8 5 2
3 6 9    ->    9 6 3
```

最终就得到了顺时针旋转 `90` 度的结果。

---

## 解法一：对角线翻转 + 每行反转

### Java 代码

```java
class Solution {
    public void rotate(int[][] matrix) {
        int n = matrix.length;

        // 1. 沿主对角线翻转
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                int temp = matrix[i][j];
                matrix[i][j] = matrix[j][i];
                matrix[j][i] = temp;
            }
        }

        // 2. 每一行左右反转
        for (int i = 0; i < n; i++) {
            int left = 0;
            int right = n - 1;
            while (left < right) {
                int temp = matrix[i][left];
                matrix[i][left] = matrix[i][right];
                matrix[i][right] = temp;
                left++;
                right--;
            }
        }
    }
}
```

---

## 执行过程演示

以：

```text
1 2 3
4 5 6
7 8 9
```

为例。

### 1）主对角线翻转

交换：

- `(0,1)` 和 `(1,0)`
- `(0,2)` 和 `(2,0)`
- `(1,2)` 和 `(2,1)`

得到：

```text
1 4 7
2 5 8
3 6 9
```

### 2）每一行反转

第一行：

```text
1 4 7 -> 7 4 1
```

第二行：

```text
2 5 8 -> 8 5 2
```

第三行：

```text
3 6 9 -> 9 6 3
```

最终结果：

```text
7 4 1
8 5 2
9 6 3
```

## `temp` 是什么意思

`temp` 是 **临时变量**，作用是交换两个数时，先把其中一个值暂存起来，避免被覆盖。

例如这三句：

```java
int temp = matrix[i][j];
matrix[i][j] = matrix[j][i];
matrix[j][i] = temp;
```

假设：

```text
matrix[i][j] = 2
matrix[j][i] = 4
```

执行过程：

1. `temp = 2`
2. `matrix[i][j] = 4`
3. `matrix[j][i] = 2`

这样两个位置的值就交换完成了。

如果没有 `temp`，你直接写：

```java
matrix[i][j] = matrix[j][i];
matrix[j][i] = matrix[i][j];
```

那第二句里的 `matrix[i][j]` 已经不是原来的值了，原值会丢失。

---

## 它是怎么遍历的

转置代码是：

```java
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        int temp = matrix[i][j];
        matrix[i][j] = matrix[j][i];
        matrix[j][i] = temp;
    }
}
```

### 为什么 `j = i + 1`

因为只需要交换 **主对角线右上角** 的元素和左下角对应元素。

主对角线上的元素不用动：

```text
(0,0) (1,1) (2,2) ...
```

而且一对元素只交换一次。

### 以 `3 x 3` 为例

原矩阵：

```text
1 2 3
4 5 6
7 8 9
```

#### 当 `i = 0`

- `j = 1`，交换 `(0,1)` 和 `(1,0)`，也就是交换 `2` 和 `4`
- `j = 2`，交换 `(0,2)` 和 `(2,0)`，也就是交换 `3` 和 `7`

矩阵变成：

```text
1 4 7
2 5 6
3 8 9
```

#### 当 `i = 1`

- `j = 2`，交换 `(1,2)` 和 `(2,1)`，也就是交换 `6` 和 `8`

矩阵变成：

```text
1 4 7
2 5 8
3 6 9
```

#### 当 `i = 2`

- `j` 已经没值了，不需要再交换

所以转置结束。

### 如果 `j` 从 `0` 开始会怎样

比如：

- `(0,1)` 和 `(1,0)` 交换一次
- 到后面 `(1,0)` 和 `(0,1)` 又会交换一次

这样就等于换了两遍，又变回去了。

所以必须让 `j` 从 `i + 1` 开始。

## 时间复杂度

- 主对角线翻转：`O(n^2)`
- 每一行反转：`O(n^2)`

总时间复杂度：

```text
O(n^2)
```

---

## 空间复杂度

只用了若干临时变量，没有额外开辟矩阵：

```text
O(1)
```

---

## 时间复杂度和空间复杂度怎么求

很多同学不是不会写代码，而是不知道复杂度怎么数。  
这题可以用“看循环执行了多少次”和“看额外开了多少空间”来判断。

### 时间复杂度怎么求

代码分两部分：

#### 第一部分：转置

```java
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        ...
    }
}
```

看每一层执行次数：

- 当 `i = 0`，`j` 会跑 `n - 1` 次
- 当 `i = 1`，`j` 会跑 `n - 2` 次
- 当 `i = 2`，`j` 会跑 `n - 3` 次
- ...
- 当 `i = n - 1`，`j` 会跑 `0` 次

总次数就是：

```text
(n - 1) + (n - 2) + ... + 1
```

这是一个等差数列，和为：

```text
n(n - 1) / 2
```

把常数和低阶项去掉后，复杂度就是：

```text
O(n^2)
```

#### 第二部分：每一行反转

```java
for (int i = 0; i < n; i++) {
    int left = 0;
    int right = n - 1;
    while (left < right) {
        ...
        left++;
        right--;
    }
}
```

外层有 `n` 行。

每一行的 `while` 循环大约执行 `n / 2` 次，因为左右指针往中间走，每次缩掉两个位置。

所以总次数大约是：

```text
n * (n / 2)
```

去掉常数后还是：

```text
O(n^2)
```

#### 两部分加起来怎么算

总时间复杂度是：

```text
O(n^2) + O(n^2)
```

复杂度里相加时，只保留最高阶，所以结果还是：

```text
O(n^2)
```

你可以简单理解为：

- 虽然做了两遍
- 但每一遍都是“扫描矩阵这一量级”
- 所以最终还是 `O(n^2)`，不是 `O(n^4)`

---

### 空间复杂度怎么求

空间复杂度看的是：除了输入的 `matrix` 之外，你额外用了多少空间。

这题里只额外用了几个变量：

```java
int n = matrix.length;
int temp = ...;
int left = 0;
int right = n - 1;
```

这些变量的个数不会随着 `n` 变大而变多，它们始终就是固定几个变量。

所以额外空间是常数级别：

```text
O(1)
```

### 为什么不是 `O(n^2)` 空间

因为我们没有新建另一个矩阵。

如果你写的是暴力解法：

```java
int[][] ans = new int[n][n];
```

那么这个新矩阵需要 `n * n` 个空间，空间复杂度才是：

```text
O(n^2)
```

而当前这道题的标准原地解法，没有开这个矩阵，所以空间复杂度是 `O(1)`。

---

## 判断复杂度的小技巧

### 看时间复杂度

重点看：

1. 循环跑了多少次
2. 每次循环里面是不是常数操作

这题里交换元素是常数操作，所以主要看循环次数。

### 看空间复杂度

重点看：

1. 有没有新开数组、集合、矩阵
2. 新开的空间是不是随着 `n` 增长

这题只有几个普通变量，所以是 `O(1)`。

---

## 易错点

### 1. 对角线翻转时，`j` 要从 `i + 1` 开始

如果从 `0` 开始，会把同一对元素交换两次，又交换回去了。

正确写法：

```java
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        ...
    }
}
```

### 2. 题目要求原地修改，不能新建结果矩阵

下面这种虽然容易想，但不符合题意：

```java
int[][] ans = new int[n][n];
```

### 3. 注意是顺时针，不是逆时针

- 顺时针：先转置，再每行反转
- 逆时针：先转置，再每列反转

不要记混。

---

## 面试怎么讲

可以这样说：

> 这题要求原地将 `n x n` 矩阵顺时针旋转 `90` 度。  
> 直接按照坐标变化搬运元素容易覆盖原值，所以我把操作拆成两步：  
> 先沿主对角线转置，让行列互换；  
> 再把每一行左右反转。  
> 两步组合起来，效果就是顺时针旋转 `90` 度。  
> 时间复杂度 `O(n^2)`，空间复杂度 `O(1)`。

---

## 拓展

### 如果是逆时针旋转 `90` 度怎么办？

方法：

1. 先沿主对角线翻转
2. 再把每一列上下翻转

### 如果不是原地要求怎么办？

那就可以直接开新矩阵，根据公式：

```text
ans[j][n - 1 - i] = matrix[i][j]
```

这样写更直观，但空间复杂度变成 `O(n^2)`。

---

## 暴力解法

有，最直观的暴力解法就是 **新建一个矩阵**。

思路：

- 遍历原矩阵每个位置 `(i, j)`
- 放到新矩阵的 `(j, n - 1 - i)` 位置

代码：

```java
class Solution {
    public void rotate(int[][] matrix) {
        int n = matrix.length;
        int[][] ans = new int[n][n];

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                ans[j][n - 1 - i] = matrix[i][j];
            }
        }

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                matrix[i][j] = ans[i][j];
            }
        }
    }
}
```

特点：

- 好理解
- 好写
- 但用了额外矩阵，不符合题目“原地”的要求

---

## 其他解法：四个位置一组直接交换

除了“转置 + 每行反转”，还可以每次处理四个位置。

例如一个元素会按照顺时针方向这样轮转：

```text
(i, j)
(j, n - 1 - i)
(n - 1 - i, n - 1 - j)
(n - 1 - j, i)
```

把这四个位置的值一次性交换。

代码：

```java
class Solution {
    public void rotate(int[][] matrix) {
        int n = matrix.length;

        for (int i = 0; i < n / 2; i++) {
            for (int j = 0; j < (n + 1) / 2; j++) {
                int temp = matrix[i][j];
                matrix[i][j] = matrix[n - 1 - j][i];
                matrix[n - 1 - j][i] = matrix[n - 1 - i][n - 1 - j];
                matrix[n - 1 - i][n - 1 - j] = matrix[j][n - 1 - i];
                matrix[j][n - 1 - i] = temp;
            }
        }
    }
}
```

特点：

- 也是原地
- 空间复杂度也是 `O(1)`
- 但是坐标更绕，初学阶段不如“转置 + 每行反转”好理解

所以面试和刷题里，一般优先掌握“转置 + 每行反转”。

---

## 本题总结

这题是一个非常经典的 **二维数组 / 矩阵变换** 题。

你要重点记住：

1. 顺时针旋转 `90` 度 = **转置 + 每行反转**
2. 坐标变化规律：`(i, j) -> (j, n - 1 - i)`
3. 原地操作时，优先考虑把复杂变换拆成几个简单变换
