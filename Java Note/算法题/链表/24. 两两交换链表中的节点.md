# 24. 两两交换链表中的节点

## 题目

给你一个链表，两两交换其中相邻的节点，并返回交换后链表的头节点。

要求：

- 必须真正交换节点
- 不能只交换节点值

例如：

```text
1 -> 2 -> 3 -> 4
```

交换后：

```text
2 -> 1 -> 4 -> 3
```

---

## 题意理解

这题的意思就是：

- 每两个节点看成一组
- 把这一组里的前后顺序交换

例如：

- `1 -> 2` 变成 `2 -> 1`
- `3 -> 4` 变成 `4 -> 3`

如果最后剩一个节点，没有配对，就保持不动。

例如：

```text
1 -> 2 -> 3
```

交换后：

```text
2 -> 1 -> 3
```

---

## 这题可以分成三种方法

1. 迭代
2. 递归
3. 压栈

其中：

- 最推荐的是迭代
- 最好理解的是递归
- 压栈能做，但不是最优雅的做法

---

## 方法一：迭代

### 核心思路

引入一个虚拟头节点 `dummy`，统一处理头节点交换问题。

当前局部结构看成：

```text
pre -> first -> second -> next
```

目标是变成：

```text
pre -> second -> first -> next
```

所以每一轮只做三步：

1. `first.next = second.next`
2. `second.next = first`
3. `pre.next = second`

交换后再让：

```java
pre = first;
```

继续处理下一组。

---

### 标准代码

```java
class Solution {
    public ListNode swapPairs(ListNode head) {
        ListNode dummy = new ListNode(0);
        dummy.next = head;

        ListNode pre = dummy;

        while (pre.next != null && pre.next.next != null) {
            ListNode first = pre.next;
            ListNode second = first.next;

            first.next = second.next;
            second.next = first;
            pre.next = second;

            pre = first;
        }

        return dummy.next;
    }
}
```

---

### 图解

假设当前局部是：

```text
pre -> first -> second -> next
```

例如：

```text
dummy -> 1 -> 2 -> 3 -> 4
```

这里：

- `pre = dummy`
- `first = 1`
- `second = 2`
- `next = 3`

#### 第一步

```java
first.next = second.next;
```

变成：

```text
1 -> 3
2
```

也就是先让 `1` 跳过 `2`，接到 `3`。

#### 第二步

```java
second.next = first;
```

变成：

```text
2 -> 1 -> 3
```

#### 第三步

```java
pre.next = second;
```

整段接回去：

```text
dummy -> 2 -> 1 -> 3 -> 4
```

然后：

```java
pre = first;
```

也就是让 `pre` 来到 `1`，准备处理下一组 `3 -> 4`。

---

### 为什么这是最优解

- 每个节点只访问常数次
- 只用了几个指针变量
- 没有递归栈
- 没有额外容器

所以：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

---

### 迭代版的常见变形

你之前见过的这些写法：

- `pre / first / second`
- `cur / firstNode / thirdNode`
- `a / b / tmp`

本质上都属于迭代版，只是变量命名和指针写法不同。

---

## 方法二：递归

### 核心思路

每次只处理前两个节点：

- 如果不足两个，直接返回
- 如果有两个，就先交换这两个
- 剩下的部分递归去处理

例如：

```text
1 -> 2 -> 3 -> 4
```

可以理解成：

1. 先处理 `1` 和 `2`
2. 再递归处理 `3 -> 4`

---

### 代码

```java
class Solution {
    public ListNode swapPairs(ListNode head) {
        if (head == null || head.next == null) {
            return head;
        }

        ListNode second = head.next;
        head.next = swapPairs(second.next);
        second.next = head;

        return second;
    }
}
```

---

### 图解

假设当前链表是：

```text
1 -> 2 -> 3 -> 4
```

当前层里：

- `head = 1`
- `second = 2`

先执行：

```java
head.next = swapPairs(second.next);
```

也就是递归处理：

```text
3 -> 4
```

递归处理完返回：

```text
4 -> 3
```

于是此时链表变成：

```text
1 -> 4 -> 3
```

然后执行：

```java
second.next = head;
```

就变成：

```text
2 -> 1 -> 4 -> 3
```

最后返回新的头节点：

```java
return second;
```

---

### 递归版的特点

优点：

- 代码短
- 思路优美
- 很适合理解“先处理后半段，再接回前两个”

缺点：

- 有递归调用栈

所以：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

---

## 方法三：压栈

### 核心思路

每次取两个节点放进栈里。

因为栈是后进先出：

- 先放第一个节点
- 再放第二个节点

弹出来时顺序就反过来了，于是就实现交换。

例如：

```text
1 -> 2
```

入栈顺序：

```text
1, 2
```

出栈顺序：

```text
2, 1
```

---

### 代码

```java
class Solution {
    public ListNode swapPairs(ListNode head) {
        if (head == null || head.next == null) {
            return head;
        }

        Stack<ListNode> stack = new Stack<ListNode>();
        ListNode p = new ListNode(-1);
        ListNode cur = head;
        head = p;

        while (cur != null && cur.next != null) {
            stack.add(cur);
            stack.add(cur.next);

            cur = cur.next.next;

            p.next = stack.pop();
            p = p.next;
            p.next = stack.pop();
            p = p.next;
        }

        if (cur != null) {
            p.next = cur;
        } else {
            p.next = null;
        }

        return head.next;
    }
}
```

---

### 图解

假设当前是：

```text
1 -> 2 -> 3 -> 4
```

第一轮把：

- `1`
- `2`

压栈。

栈顶在上面可以理解成：

```text
2
1
```

弹出时：

1. 先弹 `2`
2. 再弹 `1`

于是接回链表后就变成：

```text
2 -> 1
```

再继续处理 `3 -> 4`。

---

### 压栈版的特点

优点：

- 思路直观
- 如果你先想到“反转顺序”，这个方法比较自然

缺点：

- 本题本来只改几个指针就能做，压栈有点绕
- 面试官一般更希望你直接写链表指针交换

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

这里虽然用了栈，但每次只压 2 个节点，栈最大大小始终是常数，所以额外空间仍然是 `O(1)`。

---

## 三种方法对比

### 方法一：迭代

- 最优解
- 面试最推荐
- 时间复杂度 `O(n)`
- 空间复杂度 `O(1)`

### 方法二：递归

- 更容易从代码结构上理解
- 写法简洁
- 时间复杂度 `O(n)`
- 空间复杂度 `O(n)`

### 方法三：压栈

- 能做对
- 思路也直观
- 但不是最本质的链表解法
- 时间复杂度 `O(n)`
- 空间复杂度 `O(1)`

---

## 做题建议

这题建议你这样掌握：

1. 先掌握迭代版
2. 再理解递归版
3. 最后把压栈版当扩展

最该背的是迭代版，因为它：

- 最稳
- 最符合面试预期
- 最能训练链表指针能力

---

## 本题总结

这题就记住三种方法：

1. 迭代
2. 递归
3. 压栈

其中最核心的是迭代版局部变化：

```text
pre -> first -> second -> next
```

变成：

```text
pre -> second -> first -> next
```

只要你把这个局部交换想清楚，这题就通了。
