# 92. 反转链表 II

## 题目

给你单链表的头指针 `head` 和两个整数 `left` 和 `right`，其中 `left <= right`。请你反转从位置 `left` 到位置 `right` 的链表节点，返回反转后的链表。

要求：

- 一次遍历完成
- 位置编号从 1 开始

例如：

```text
head = [1, 2, 3, 4, 5], left = 2, right = 4
```

反转后：

```text
[1, 4, 3, 2, 5]
```

---

## 题意理解

这题是「206. 反转链表」的升级版。

区别在于：

- 反转链表 I：反转整个链表
- 反转链表 II：只反转链表中的某一段，其余部分保持不变

核心要解决的问题：

1. 怎么定位到要反转的那一段
2. 怎么只反转这一段，不破坏前后连接
3. 怎么处理好边界的连接

---

## 这题可以分成三种方法

1. **一次遍历 + 头插法（最优解）**
2. 截断 + 反转 + 拼接
3. 转换成数组操作

其中：

- 最推荐的是 **一次遍历 + 头插法**
- 最容易理解的是 **截断 + 反转 + 拼接**
- 最暴力的是 **转换成数组操作**

---

## 方法一：一次遍历 + 头插法（最优解）

### 核心思路

先找到反转区间的前一个节点 `pre`，然后从 `pre.next` 开始，重复执行：

- 把 `pre.next` 后面的节点摘出来
- 插到 `pre` 的后面（头插法）

重复 `right - left` 次，就完成了区间反转。

### 关键点

- 引入 `dummy` 虚拟头节点，统一处理 `left = 1` 的情况
- 先走 `left - 1` 步，定位到 `pre`
- 用三个指针完成反转：`pre`、`cur`、`next`

### 标准代码

```java
class Solution {
    public ListNode reverseBetween(ListNode head, int left, int right) {
        ListNode dummy = new ListNode(0);
        dummy.next = head;

        // 1. 走到 left 的前一个节点
        ListNode pre = dummy;
        for (int i = 0; i < left - 1; i++) {
            pre = pre.next;
        }

        // 2. 头插法反转区间
        ListNode cur = pre.next;
        for (int i = 0; i < right - left; i++) {
            ListNode next = cur.next;
            cur.next = next.next;
            next.next = pre.next;
            pre.next = next;
        }

        return dummy.next;
    }
}
```

---

### 图解

以：

```text
head = [1, 2, 3, 4, 5], left = 2, right = 4
```

为例。

#### 第一步：定位到 `pre`

```java
for (int i = 0; i < left - 1; i++) {
    pre = pre.next;
}
```

`left - 1 = 1`，所以 `pre` 走 1 步，来到节点 `1`：

```text
dummy -> 1 -> 2 -> 3 -> 4 -> 5
         ^
        pre
```

此时：

- `pre = 1`
- `cur = pre.next = 2`

#### 第二步：开始头插法

要反转的区间是 `2 -> 3 -> 4`，需要交换 `right - left = 2` 次。

##### 第 1 次（把 3 插到 pre 后面）

初始状态：

```text
pre=1 -> cur=2 -> 3 -> 4 -> 5
```

```java
ListNode next = cur.next;  // next = 3
cur.next = next.next;      // 2 -> 4
next.next = pre.next;      // 3 -> 2
pre.next = next;           // 1 -> 3
```

变化过程：

```text
1 -> 2 -> 3 -> 4 -> 5       初始
1 -> 2 -> 4 -> 5            执⾏ cur.next = next.next，3 被摘出来
      \_ 3 _/               
1 -> 3 -> 2 -> 4 -> 5       执⾏ next.next = pre.next 和 pre.next = next
```

结果：

```text
1 -> 3 -> 2 -> 4 -> 5
```

##### 第 2 次（把 4 插到 pre 后面）

此时：

- `pre = 1`（不变）
- `cur = 2`（不变，cur 一直指向最初的那个节点）

```java
ListNode next = cur.next;  // next = 4
cur.next = next.next;      // 2 -> 5
next.next = pre.next;      // 4 -> 3
pre.next = next;           // 1 -> 4
```

变化过程：

```text
1 -> 3 -> 2 -> 4 -> 5       当前
1 -> 3 -> 2 -> 5            执⾏ cur.next = next.next，4 被摘出来
           \_ 4 _/
1 -> 4 -> 3 -> 2 -> 5       执⾏ next.next = pre.next 和 pre.next = next
```

最终结果：

```text
1 -> 4 -> 3 -> 2 -> 5
```

反转完成。

---

### 为什么 cur 不需要移动

这是一个非常重要的理解点。

很多人一开始会困惑：为什么 `cur` 从不更新？每次循环都还是同一个 `cur`？

原因在于：

- `pre` 始终指向反转区间的前一个节点
- `cur` 始终指向反转区间的**第一个节点**
- 每次循环都是把 `cur.next`（即当前区间的第二个节点）**摘出来**，头插到 `pre` 后面
- 所以 `cur` 的位置一直在「往右退」，它的 `next` 不断变成新的节点

举个例子你就明白了：

```text
初始：  1 -> 2 -> 3 -> 4 -> 5
          pre  cur
               
第1次： 1 -> 3 -> 2 -> 4 -> 5
          pre      cur
               
第2次： 1 -> 4 -> 3 -> 2 -> 5
          pre           cur
```

看到没？**cur 虽然没有动，但因为它后面的节点不断地被头插到前面去，所以 cur 实际上在相对地往后移动**。

这恰好就是头插法的精妙之处：你不需要一个指针跟着 cur 往后走，因为每一次头插都自动把 cur 往后推了一位。

### 复杂度

- 时间复杂度：`O(n)`，只遍历了一次
- 空间复杂度：`O(1)`，只用了几个指针变量

## 方法二：截断 + 反转 + 拼接

### 核心思路

思路更直接：

1. 先找到反转区间的前一个节点 `pre` 和后一个节点 `nextPart`
2. 把反转区间单独截出来
3. 反转这个子链表
4. 再拼接回去

### 代码

```java
class Solution {
    public ListNode reverseBetween(ListNode head, int left, int right) {
        ListNode dummy = new ListNode(0);
        dummy.next = head;

        // 1. 找到 left 的前一个节点
        ListNode pre = dummy;
        for (int i = 0; i < left - 1; i++) {
            pre = pre.next;
        }

        // 2. 找到 right 节点
        ListNode rightNode = pre;
        for (int i = 0; i < right - left + 1; i++) {
            rightNode = rightNode.next;
        }

        // 3. 记录 right 后面的部分
        ListNode nextPart = rightNode.next;

        // 4. 截断出反转区间
        ListNode leftNode = pre.next;
        rightNode.next = null;

        // 5. 反转子链表
        reverseList(leftNode);

        // 6. 拼接回去
        pre.next = rightNode;
        leftNode.next = nextPart;

        return dummy.next;
    }

    private ListNode reverseList(ListNode head) {
        ListNode pre = null;
        ListNode cur = head;
        while (cur != null) {
            ListNode next = cur.next;
            cur.next = pre;
            pre = cur;
            cur = next;
        }
        return pre;
    }
}
```

### 图解

以：

```text
[1, 2, 3, 4, 5], left = 2, right = 4
```

为例。

#### 找出四个关键节点

```text
dummy -> 1 -> 2 -> 3 -> 4 -> 5 -> null
         ^         ^         ^    ^
        pre      leftNode  right  nextPart
```

#### 截断

把 `rightNode.next` 置为 `null`，得到两个链表：

```text
反转区间： 2 -> 3 -> 4 -> null
剩余部分： dummy -> 1 -> null（pre 连着这个）
            5 -> null（nextPart）
```

#### 反转子链表

调用 `reverseList(2)`，得到：

```text
4 -> 3 -> 2 -> null
```

#### 拼接

```java
pre.next = rightNode;    // 1 -> 4
leftNode.next = nextPart; // 2 -> 5
```

最终：

```text
1 -> 4 -> 3 -> 2 -> 5
```

### 优缺点

优点：

- 思路非常直观，容易理解
- 复用了标准反转链表的方法

缺点：

- 遍历了两次链表（一次找 pre，一次找 rightNode）
- 严格来说不是「一次遍历」

复杂度：

- 时间复杂度：`O(n)`，虽然遍历了两次，但总步数还是 `O(n)`
- 空间复杂度：`O(1)`

## 方法三：转换成数组操作

### 核心思路

把链表转成数组，在数组里反转指定区间，再重新构建链表。

### 代码

```java
class Solution {
    public ListNode reverseBetween(ListNode head, int left, int right) {
        // 1. 链表转数组
        List<Integer> list = new ArrayList<>();
        ListNode cur = head;
        while (cur != null) {
            list.add(cur.val);
            cur = cur.next;
        }

        // 2. 反转数组中的区间
        int l = left - 1, r = right - 1;
        while (l < r) {
            int temp = list.get(l);
            list.set(l, list.get(r));
            list.set(r, temp);
            l++;
            r--;
        }

        // 3. 数组重新构造成链表
        ListNode dummy = new ListNode(0);
        ListNode p = dummy;
        for (int val : list) {
            p.next = new ListNode(val);
            p = p.next;
        }

        return dummy.next;
    }
}
```

### 优缺点

优点：

- 最容易想到
- 逻辑最直接
- 不容易出错

缺点：

- 额外用了 `O(n)` 的空间
- 如果面试官要求「就地反转」或者「一次遍历」，这个解法不达标

复杂度：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 易错点

### 1. 位置编号从 1 开始

`left` 和 `right` 是 1-indexed，所以定位 `pre` 需要走 `left - 1` 步，而不是 `left` 步。

### 2. 头插法的循环次数是 `right - left`，不是 `right - left + 1`

因为：

- 区间长度为 `right - left + 1`
- 第一个节点不需要插，从第二个节点开始，每个节点需要头插一次
- 所以循环次数 = `right - left`

### 3. `left = 1` 时的边界

如果不加 `dummy` 节点，`left = 1` 时 `pre` 没有前一个节点，需要单独处理。

所以统一加 `dummy` 是最稳的做法。

### 4. `cur` 不移动的理解

头插法里 `cur` 指向的始终是反转区间的第一个节点，不需要移动。每次头插 `cur.next` 后，`cur` 的 next 会自动指向下一个待处理的节点。记住：**cur 不动，动的是 cur.next**。

## 面试怎么讲

可以这样说：

> 这题是反转链表的区间版本。我采用一次遍历 + 头插法来做。
>
> 先引入一个 dummy 节点，统一处理 left = 1 的边界情况。
>
> 然后走 left - 1 步，定位到反转区间的前一个节点 pre。
>
> 接下来执行 right - left 次头插：每次把 cur.next 这个节点摘出来，插入到 pre 的后面。
>
> 这样做的好处是只遍历一次，空间复杂度 O(1)，也是面试官最期望看到的解法。

---

## 本题总结

这题的核心就一句话：

**用头插法，把区间内的每个节点依次插到 pre 后面。**

```text
pre 固定不动，cur 始终指向区间第一个节点
每次把 cur.next 摘出来，头插到 pre.next
重复 right - left 次
```

记住这三个关键点，这题就通了：

1. `dummy` 统一处理边界
2. `pre` 走 `left - 1` 步
3. 循环 `right - left` 次的头插
