# 142. 环形链表 II

## 题目

给定一个链表的头节点 `head`，判断链表中是否有环，并返回链表开始入环的第一个节点。

要求：

- 如果链表无环，返回 `null`
- 不能修改链表

**示例 1：**

```text
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的链表节点
解释：链表中有一个环，其尾部连接到第二个节点
```

```
3 -> 2 -> 0 -> -4
     ^           |
     |           v
     +-----<-----+
```

**示例 2：**

```text
输入：head = [1,2], pos = 0
输出：返回索引为 0 的链表节点
解释：链表中有一个环，其尾部连接到第一个节点
```

```
1 -> 2
^    |
+----+
```

---

## 题意理解

这题是「[[141. 环形链表]]」的进阶版。141 只问有没有环，142 不仅要判断有环，还要**找到环的入口节点**。

换句话说：

- 如果链表没有环 → 返回 `null`
- 如果有环 → 返回环开始的那个节点

例如：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7 -> 8
               ^              |
               |              v
               10 <- 9 <- 8 <-
```

这里的环是 `4→5→6→7→8→9→10→4`，**入口节点（环开始的地方）就是 4**。

---

## 这题可以分成两种方法

1. **哈希表法（HashSet）** — 最容易想到
2. **Floyd 判圈算法（快慢指针）** — 最优解

其中：

- 最推荐的是 Floyd 快慢指针，空间复杂度 `O(1)`
- 最好理解的是哈希表法

---

## 方法一：哈希表法（HashSet）

### 核心思路

遍历链表，每经过一个节点就用 HashSet 记录下来：

- 如果当前节点已经在 HashSet 中 → 这个节点就是环的入口
- 如果遍历到 `null` → 说明链表无环

为什么 HashSet 能找出入口？因为入环后一定会第二次访问到入口节点，第一次重复就是我们要找的答案。

---

### 代码

```java
import java.util.HashSet;

public class Solution {
    public ListNode detectCycle(ListNode head) {
        HashSet<ListNode> seen = new HashSet<>();

        ListNode cur = head;
        while (cur != null) {
            if (seen.contains(cur)) {
                return cur;  // 第一次重复的节点就是环入口
            }
            seen.add(cur);
            cur = cur.next;
        }

        return null;  // 遍历完没有重复 → 无环
    }
}
```

---

### 图解

以链表 `1→2→3→4→5→6→7→8→3` 为例：

```
HashSet: {}

1 → 不在集合中，加入
HashSet: {1}

2 → 不在集合中，加入
HashSet: {1, 2}

3 → 不在集合中，加入
HashSet: {1, 2, 3}

4 → 不在集合中，加入
HashSet: {1, 2, 3, 4}

...继续遍历...

当 cur 再次回到 3 时：
3 → 已经在集合中！！！ → 返回 3
```

---

### 复杂度

- 时间复杂度：`O(n)` — 每个节点最多访问一次
- 空间复杂度：`O(n)` — 需要存储所有访问过的节点

## 方法二：Floyd 判圈算法（快慢指针）

### 核心思路

这是本题的**最优解**，分为两个阶段：

**阶段一：判断是否有环**

- `slow` 每次走 1 步
- `fast` 每次走 2 步
- 如果 `slow == fast` → 有环
- 如果 `fast` 走到 `null` → 无环

**阶段二：找到环入口**

- `slow` 和 `fast` 相遇后，将 `slow`（或 `fast`）重新指向 `head`
- 然后两个指针**都每次走 1 步**
- 再次相遇的位置就是**环的入口节点**

---

### 数学证明

定义：

```
a = 从头节点到环入口的距离（节点数）
b = 从环入口到相遇点的距离
c = 从相遇点回到环入口的距离
环的长度 = b + c
```

当 slow 和 fast 相遇时：

- `slow` 走了 `a + b` 步
- `fast` 走了 `a + b + n(b + c)` 步（n 是 fast 在环内多走的圈数）
- 因为 `fast` 速度是 `slow` 的 2 倍：`2(a + b) = a + b + n(b + c)`

化简得：

```
a + b = n(b + c)
a = n(b + c) - b
a = (n - 1)(b + c) + c
```

当 `n = 1` 时（最简单情况），`a = c`。

**结论**：从头节点到入口的距离 `a`，等于从相遇点继续走到入口的距离 `c`。

所以当两个指针分别从头节点和相遇点、以相同速度前进时，它们一定在环入口相遇。

---

### 代码

```java
public class Solution {
    public ListNode detectCycle(ListNode head) {
        ListNode slow = head;
        ListNode fast = head;

        // 阶段一：检测是否有环
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;

            if (slow == fast) {
                // 阶段二：找到环入口
                slow = head;
                while (slow != fast) {
                    slow = slow.next;
                    fast = fast.next;
                }
                return slow;
            }
        }

        return null;  // 无环
    }
}
```

---

### 图解

以链表 `1→2→3→4→5→6→7→8→3` 为例：

```
环入口: 3
a = 2 (1→2→3)
b = 4 (3→4→5→6→7)
c = 2 (7→8→3)
环长度 = b + c = 6
```

**阶段一：快慢指针追逐**

```text
步数  slow  fast    说明
────────────────────────────
 0     1     1     起点
 1     2     3
 2     3     5     slow 刚到入口，fast 已在环内
 3     4     7
 4     5     3     fast 进入第二圈
 5     6     5
 6     7     7     ★ slow == fast = 7，相遇！
```

**阶段二：找环入口**

相遇在节点 7，此时：

1. 将 `slow` 重置到 head（节点 1）
2. `slow` 和 `fast` 都每次走 1 步

```text
步数  slow  fast    说明
────────────────────────
 0     1     7     重置 slow 到头
 1     2     8
 2     3     3     ★ slow == fast = 3，环入口！
```

验证：

- `slow` 从 head 走了 2 步到达入口 `3`（a = 2）
- `fast` 从相遇点 `7` 走了 2 步到达入口 `3`（c = 2）
- ✅ `a = c`

---

### 复杂度

- 时间复杂度：`O(n)` — slow 和 fast 各走 O(n) 步
- 空间复杂度：`O(1)` — 只用了两个指针

## 做题建议

1. **先掌握 Floyd 快慢指针** — 这是本题的面试目标解法
2. **理解数学证明** — 不需要背公式，但要理解 a = c 的推导逻辑
3. **再掌握 HashSet 法** — 作为兜底，面试时如果 Floyd 写不出来，至少能给出可工作的解
4. **画图验证** — 随手画一个带环链表，手动模拟 Floyd 两阶段过程，比死记硬背有效得多

---

## 易错点

1. **忘记处理无环情况** — `while` 条件要检查 `fast != null && fast.next != null`，缺少任何一个都可能 NPE
2. **阶段二重置指针后忘记都走 1 步** — 重置后两个指针都要走 1 步，不要再快慢不同了
3. **混淆「相遇点」和「入口点」** — 第一次相遇的位置不是入口，需要第二阶段才能找到
4. **链表只有两个节点成环** — 例如 `1→2→1`，需要确保算法能正确处理
5. **返回的是节点，不是节点值或索引** — 题目要求返回 `ListNode`，不是 `int`

---

## 本题总结

LeetCode 142 是链表环问题的经典题，核心就是 **Floyd 判圈算法的两阶段应用**：

1. **阶段一**：快慢指针检测是否有环
2. **阶段二**：从头节点和相遇点同步前进，相遇处即为环入口

```text
检测环：slow走1步，fast走2步 → 相遇即有环
找入口：slow重置为head，两指针都走1步 → 再相遇即是入口
```

数学本质是证明 `a = (n-1)(b+c) + c`，当 `n=1` 时简化为 `a = c`。

对比 141 题，141 只要求检测是否有环，142 在此基础上还要找到环的入口。建议把[[141. 环形链表]]和本题一起复习。
